# Memory And Hidden Information

WorldForge tries to keep track of what happened, who knows it, and whether the player is allowed to see it.

## Facts, Claims, Rumors, And Beliefs

These are not the same:

- A fact is true in the world.
- A claim is something someone said.
- A rumor is information circulating without guaranteed truth.
- A belief is what a character thinks.
- A secret is true but not visible to the player yet.

Example:

> I tell the guard I work for the headmaster.

That can become a recorded claim. It should not automatically become a true employment record.

## What The Game Remembers

WorldForge can remember:

- Things that happened in play.
- Important lore.
- Character details.
- NPC relationships.
- Faction actions.
- Recent local events.
- Player claims and discoveries.
- Hidden facts that exist behind the scene.

The point is not perfect memory of every sentence. The point is usable memory that can come back when it matters.

## What The Player Can Know

The game tries to separate world truth from player-visible truth.

Your character can usually know things they:

- Saw directly.
- Were told by someone.
- Found in a document or object.
- Learned through a fitting ability.
- Inferred from available clues.
- Already knew from their background.

Your character should not automatically know an NPC's private plan just because the game has it stored.

## Why This Matters

In many AI roleplay tools, hidden information leaks because the narrator knows too much. WorldForge tries to prevent that by giving the narrator only the parts that should be visible.

This makes investigation, bluffing, secrets, and social play more meaningful.

## Memory Is Source-Bound

"Source-bound" means the game cares where information came from.

For example:

- Player said it.
- NPC reported it.
- Faction discovered it.
- World state confirms it.
- Lore says it.
- It is hidden backend truth.

The same sentence can be safe in one source and unsafe in another. A player can say a secret name as a guess, but the narrator should not confirm that the guess is correct unless the player has proof.

## Practical Play Tips

If you want the game to track something, make it concrete:

> I write down the symbol from the door and compare it to the mark on the courier's ring.

If you are guessing, say you are guessing:

> I suspect the priest is lying, but I know I do not have proof yet.

If you want only visible information, ask that way:

> What can I actually see from here?
