<objective>
Replace the 62-line text-file logger with a pino-backed core that preserves the `createLogger(tag)` API used by 33 existing call sites. Key design properties enforced by this plan:

1. **Field-level recursive truncation** — `serializePayload` walks objects/arrays key-by-key and truncates ONLY oversized string/blob values in place. Shape is preserved: `{ targetTags: [...], assembledChars: N, type: "oracle_result" }` survives even when `targetTags[0]` is 50 KB.
2. **`payload.*` redaction at depths 1-6** — pino.redact paths cover the `payload` wrapper at every nesting level up to depth 6 (matches `SERIALIZE_MAX_DEPTH`), so `log.info("x", { config: { providers: { judge: { transport: { headers: { auth: { apiKey: "..." } } } } } })` → `[REDACTED]` even at depth 6. Without this, secrets at depth 5-6 slip past redact.
3. **Sync append JSONL writes** — `getOrOpenTurnStream` uses `fs.appendFileSync` inside `_write`, not async `createWriteStream`. Crash-safe per Gemini+Codex consensus.
4. **Turn-id-suffixed filenames** — `turn-{tick}-{turnId.slice(0,8)}.jsonl` prevents same-campaign same-tick retry overwrites.
5. **NO SILENT DEGRADATION (per CLAUDE.md)** — pino-pretty missing → throw at startup (production); malformed persisted observability → throw with Zod issue list; side-car write failure → log.error + throw.
6. **Runtime config snapshot cache** — `getObservabilityConfigSnapshot()` is a synchronous accessor that replaces `loadSettings()` on hot paths.
7. **`GSD_LOG_ROOT` env + `resetLoggerForTest`** — no `process.chdir()`, no singleton leakage across tests. **Pretty transport is skipped in test mode** so resetLoggerForTest can rebuild pino N times without leaking N pino-pretty worker threads.
8. **`GSD_CAMPAIGNS_ROOT` env + `getCampaignsDir()` function** — replaces module-level `CAMPAIGNS_DIR` const so route-level integration tests in Plans 58-03 and 58-04 can sandbox `app.request("/api/chat/action", ...)` per-test. Without this, fixtures seeded at `${tmp}/campaigns/<id>` are invisible to the chat handler which still resolves through the hardcoded module-relative path. **All current callers are migrated** (manager.ts × 3 sites, worldbook-library/paths.ts, paths.ts itself, the campaign/index.ts barrel re-export, and the path-based test files).

  info: (message: string, data?: unknown) => void;
  warn: (message: string, data?: unknown) => void;

Call sites (from grep — DO NOT break these signatures): turn-processor, oracle, npc-agent, reflection-agent, prompt-assembler, tool-executor, faction-tools, npc-offscreen, world-engine, target-context, state-snapshot, embeddings, lore-cards, episodic-events, chat routes, etc.

Wrapping rule: callers pass `{ ...data }`; the shim wraps as `child.info({ payload: serializePayload(data) }, message)`. This means **every field of `data` ends up under `payload.*` in the final record**. Therefore pino.redact paths MUST cover `payload.*` variants at multiple depths up to `SERIALIZE_MAX_DEPTH` (6).

Field-level truncation contract:
- Object: recurse into each key; each string > 10000 chars → `{ _truncated: true, _sha256, _length, _head (first 500), _tail (last 500) }`; other types unchanged.
- Array: iterate element-by-element; same rules apply per element.
- Max recursion depth: 6. At depth 6, collapse to `{ _truncated: true, _reason: "max-depth", _typeof: typeof value }`.
- Cycle safety: `WeakSet` of visited objects; cyclic refs → `{ _truncated: true, _reason: "cycle" }`.
- Whole-object serialized size is no longer a truncation trigger. Each FIELD is checked individually.

    ```typescript
    import { createHash } from "node:crypto";

    const TRUNCATION_THRESHOLD = 10_000;
    export const SERIALIZE_MAX_DEPTH = 6;

    export interface TruncatedReference {
      _truncated: true;
      _sha256: string;
      _length: number;
      _head: string;
      _tail: string;
    }

    export function truncatedReference(s: string): TruncatedReference {
      return {
        _truncated: true,
        _sha256: createHash("sha256").update(s).digest("hex"),
        _length: s.length,
        _head: s.slice(0, 500),
        _tail: s.slice(-500),
      };
    }

    function serializeString(s: string): string | TruncatedReference {
      return s.length > TRUNCATION_THRESHOLD ? truncatedReference(s) : s;
    }

    function serializeValue(value: unknown, depth: number, seen: WeakSet<object>): unknown {
      if (depth > SERIALIZE_MAX_DEPTH) {
        return { _truncated: true, _reason: "max-depth", _typeof: typeof value };
      }
      if (value === null || value === undefined) return value;
      if (typeof value === "string") return serializeString(value);
      if (typeof value === "number" || typeof value === "boolean" || typeof value === "bigint") return value;
      if (value instanceof Error) {
        return {
          name: value.name,
          message: serializeString(value.message),
          stack: value.stack ? serializeString(value.stack) : undefined,
        };
      }
      if (Array.isArray(value)) {
        if (seen.has(value)) return { _truncated: true, _reason: "cycle" };
        seen.add(value);
        return value.map((v) => serializeValue(v, depth + 1, seen));
      }
      if (typeof value === "object") {
        if (seen.has(value as object)) return { _truncated: true, _reason: "cycle" };
        seen.add(value as object);
        const out: Record<string, unknown> = {};
        for (const [k, v] of Object.entries(value as Record<string, unknown>)) {
          out[k] = serializeValue(v, depth + 1, seen);
        }
        return out;
      }
      return String(value);
    }

    export function serializePayload(value: unknown): unknown {
      return serializeValue(value, 0, new WeakSet());
    }
    ```
