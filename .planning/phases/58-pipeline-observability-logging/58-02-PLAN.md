**Judge role semantics:**
- `withRole("oracle", ...)` — deterministic-engine logic of `executeOracleCall` (chance math, validation).
- `withRole("judge", ...)` — nested INSIDE oracle/reflection around ONLY the structured-output LLM call (generateObject). Both frames fire: `oracle.call` carries role="oracle"; `llm.attempt` emitted from inside the nested frame carries role="judge".
