  <action>
    **npc-tools.ts**: Create `createNpcAgentTools(campaignId, npcId, tick, judgeProvider)` factory using `ai` SDK `tool()` with Zod inputSchema. Four tools:

    1. `act` — inputSchema: `{ action: z.string() }`. Execute calls `callOracle()` with NPC's tags as actorTags, environment from NPC's location. On success (strong_hit/weak_hit), execute relevant state changes via `executeToolCall` (e.g., add_tag, set_relationship). On miss, log the failed attempt. Always `log_event` the action + outcome. Returns `{ oracleResult, outcome }`.

    2. `speak` — inputSchema: `{ dialogue: z.string(), target: z.string().optional() }`. No Oracle needed. Stores dialogue as episodic event via `storeEpisodicEvent` with participants=[npcName, target]. Returns `{ spoke: true, dialogue }`.

    3. `move_to` — inputSchema: `{ targetLocation: z.string() }`. Resolves target location by name (case-insensitive). Validates adjacency by checking current location's connectedTo array. On valid: updates `npcs.currentLocationId` in DB. On invalid: returns error "Not adjacent". Returns `{ moved: true, from, to }`.

    4. `update_own_goal` — inputSchema: `{ oldGoal: z.string(), newGoal: z.string(), type: z.enum(["short_term","long_term"]) }`. Reads NPC's goals JSON, finds and replaces oldGoal with newGoal (or appends if oldGoal not found). Writes back to DB. Returns `{ updated: true, goals }`.

    **npc-agent.ts**: Create two exported functions:

    `tickNpcAgent(campaignId, npcId, tick, judgeProvider)` — Core per-NPC tick:
    1. Load NPC from DB (name, persona, tags, goals, beliefs, currentLocationId)
    2. Load NPC's current location (name, description, tags)
    3. Load nearby entities (other NPCs and player at same location, items at location)
    4. Search episodic events involving this NPC (use searchEpisodicEvents with NPC name as query, limit 3) — skip if embedder not available
    5. Load relationship graph for this NPC via getRelationshipGraph
    6. Build NPC agent system prompt: persona, goals, beliefs, location context, nearby entities, recent memories, relationships
    7. Call `generateText()` with Judge role, NPC tools, temperature 0, stopWhen stepCountIs(2)
    8. Return array of tool call results for logging

    `tickPresentNpcs(campaignId, tick, judgeProvider, playerLocationId)` — Orchestrator:
    1. Query DB for all NPCs where tier='key' AND currentLocationId=playerLocationId
    2. Run tickNpcAgent sequentially for each (not Promise.all — avoid conflicts)
    3. Collect and return all NPC action results
    4. Wrap each tick in try/catch — log failures, continue to next NPC

    NPC agent system prompt template (Claude's discretion on wording):
    ```
    You are {name}, a character in a text RPG world.
    Your persona: {persona}
    Your traits: {tags}
    Your goals: {goals}
    Your beliefs: {beliefs}
    You are at: {location.name} — {location.description}
    Also here: {nearbyEntities}
    Recent events involving you: {episodicMemories}
    Your relationships: {relationships}

    Decide your next action. You may:
    - act(action) — attempt an action (will be evaluated for success)
    - speak(dialogue) — say something to someone present
    - move_to(location) — travel to an adjacent location
    - update_own_goal(old, new) — revise your goals based on events

    Choose ONE action that best serves your current goals. If nothing pressing, you may pass (call no tools).
    ```
