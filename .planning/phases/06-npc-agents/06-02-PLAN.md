  <action>
    **npc-offscreen.ts**: Create `simulateOffscreenNpcs(campaignId, tick, judgeProvider, playerLocationId, interval?)` function.

    1. Check `tick % interval === 0` (interval defaults to 5). If not, return empty array immediately.
    2. Query DB for all NPCs where `tier='key'` AND `currentLocationId != playerLocationId` (off-screen Key NPCs).
    3. If none found, return empty array.
    4. For each off-screen NPC, build a summary: name, persona, tags, current location name, goals.
    5. Build a single batch prompt for the Judge LLM:
       ```
       You are the world simulation engine. For each NPC listed below, determine what they've been doing off-screen.
       For each NPC, return a structured update:
       - npcName: the NPC's name
       - newLocation: name of location they moved to (or null if staying)
       - actionSummary: brief description of what they did
       - goalProgress: any goal updates (or null if unchanged)

       NPCs:
       {foreach npc: name, persona, location, goals}
       ```
    6. Call `generateObject()` with Judge role, temperature 0, Zod schema for array of `{ npcName, newLocation, actionSummary, goalProgress }`.
    7. For each update result:
       - If newLocation: resolve location by name, update `npcs.currentLocationId`
       - If goalProgress: update `npcs.goals` JSON
       - Store action summary as episodic event via `storeEpisodicEvent`
       - Increment `npcs.inactiveTicks` (reset to 0 if NPC acted)
    8. Return array of applied updates for logging.

    Export helper `parseOffscreenUpdates` and `applyOffscreenUpdate` as named exports for testability.

    **index.ts**: Export `simulateOffscreenNpcs` from engine barrel.

    **chat.ts**: In `buildOnPostTurn`, after NPC present-ticks (added in Plan 01), add:
    ```typescript
    // Off-screen batch simulation (every N ticks)
    try {
      await simulateOffscreenNpcs(campaignId, summary.tick, judgeProvider, playerLocationId);
    } catch (err) {
      log.warn("Off-screen NPC simulation failed", err);
    }
    ```
  </action>
