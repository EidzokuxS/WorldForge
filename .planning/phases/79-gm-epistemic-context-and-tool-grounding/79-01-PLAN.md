## Tasks

### Task 1: RED Forest Outpost Prompt Leak Test

Add a deterministic test fixture where:

- current scene/location is Shibuya;
- background/offscreen roster or recent world state contains `Okutama Safe Zone - Forest Outpost`;
- raw recent conversation contains `Forest Outpost` from an older/background beat;
- GM/ScenePlanner prompt builders are exercised through exported test helpers or a model-facing packet builder.

Expected RED assertions:

- model-facing GM packet contains Shibuya local scene data;
- it does not contain `Forest Outpost`;
- it does not contain hidden/background actor labels;
- it does not contain hidden/background actor ids;
- it does not contain raw `forbiddenActorLabels`, `roster.background`, or equivalent backend-only fields;
- ScenePlanner repair prompts do not echo forbidden names from invalid candidate JSON or validation issue text;
- it exposes only counts or opaque categories for private/background state.

### Task 2: GREEN Model-Facing Scene View

Implement a small pure function that converts backend `SceneFrame` into a prompt-safe view:

- active actors and clear support actors are visible;
- hinted support is represented only through awareness hints;
- background actors and forbidden labels are removed by name/id;
- recent events are filtered to local/player-perceivable facts and redacted against forbidden/background actors and remote location terms;
- movement candidates and target candidates remain as legal affordances;
- hidden/background counts are allowed, but no hidden names.
- opaque hidden presence uses generic player-facing categories only, e.g. `unknown nearby presence`, `distant movement`, or counts; never `hidden actor: <name>`.

Add companion helpers:

- collect forbidden prompt terms from non-clear/background actor labels, ids, and remote/offscreen location names that appear in backend-only context;
- redact or drop recent conversation lines that contain forbidden actor names/ids before GM/ScenePlanner prompts;
- assert prompt safety in tests so a future raw `SceneFrame` dump fails loudly.

Prefer structural exclusion over string-only masking. The regression must prove that offscreen event content is absent, not merely that the exact literal `Forest Outpost` was replaced. Redaction is the last guard for repair/candidate text, not the primary data-flow design.

Forbidden-term matching must be case-insensitive and cover simple display-name variants that the code already exposes, but it must avoid broad semantic censorship. Conversation redaction is scoped to actor labels/ids and backend-only ids. Remote location leakage is primarily prevented by structural omission of non-local sections; do not globally censor a location name the player explicitly typed or already knows unless it arrived only from backend-private context.

Use this view in `buildGmTurnDecisionPrompt` and `buildDefaultScenePlannerPrompt`.

### Task 3: GREEN Repair Prompt Redaction

Patch ScenePlanner repair prompt construction so validation issues and invalid candidate JSON are passed through the same forbidden-term redaction before the second model call.

This is mandatory. Redacting only the first prompt is insufficient because a failed model output can echo a forbidden/background name into the repair prompt.
