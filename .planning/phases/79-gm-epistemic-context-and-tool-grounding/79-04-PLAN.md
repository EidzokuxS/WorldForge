## Tasks

### Task 1: RED E2E Wrong-Location Tool Plan Test

Add a mocked GM/ScenePlanner route or turn-processor test that attempts the exact wrong-location tool plan while the player is in Shibuya.

Expected:

- turn fails or repairs before mutation;
- snapshot restores if failure escapes the engine;
- no NPC is inserted at Forest Outpost;
- narrator does not mention the remote spawn as local scene truth.
- if a mixed ScenePlan contains one legal local mutation and one illegal remote mutation, neither mutation is committed.
- pending committed events for the failed tick are drained if the failure path queues any before abort.

### Task 2: GREEN Context/Tool Diagnostics

Add concise logs around model-facing packet construction and tool grounding:

- visible actor count;
- hidden/background count;
- local recent event count;
- allowed tool count;
- rejected grounding reason and tool name;
- no raw API keys, secrets, giant prompt dumps, or full hidden actor lists.

The diagnostic code should use reason codes such as `remote_location_ref`, `ambiguous_entity_ref`, `hidden_actor_ref`, or `unexposed_item_ref`. Avoid logging rejected hidden names when the name itself is private; include a hash/id prefix only if that is already safe in local logs.
