# Plan 23-01: Persist ipContext in Campaign Config

## Objective

Research runs ONCE during suggest-seeds, results are saved to campaign config.json, and reused by all downstream operations (generate, regenerate-section) without re-searching.

## Current State

- `suggest-seeds` → calls `researchKnownIP()` → returns ipContext in response
- `generate` → calls `researchKnownIP()` AGAIN → duplicates work
- `regenerate-section` → NO ipContext at all → loses franchise context
- Frontend stores ipContext in React state → lost on page reload
- config.json has: `{ name, premise, seeds }` — no ipContext field

## Tasks

### Task 1: Add ipContext to shared types and campaign config

<read_first>
- shared/src/types.ts
- backend/src/campaign/manager.ts
</read_first>

<action>
1. In `shared/src/types.ts`, add `IpResearchContext` interface if not already exported:
   ```typescript
   export interface IpResearchContext {
     franchise: string;
     keyFacts: string[];
     tonalNotes: string[];
     source: "mcp" | "llm";
   }
   ```

2. In `backend/src/campaign/manager.ts`, update config read/write to include optional `ipContext` field:
   - The config.json structure becomes: `{ name, premise, seeds, ipContext? }`
   - Add function `saveIpContext(campaignId: string, ipContext: IpResearchContext): void` that reads config.json, adds ipContext field, writes back
   - Add function `loadIpContext(campaignId: string): IpResearchContext | null` that reads config.json and returns ipContext or null
</action>

<acceptance_criteria>
- `shared/src/types.ts` exports `IpResearchContext` interface with franchise, keyFacts, tonalNotes, source fields
- `backend/src/campaign/manager.ts` exports `saveIpContext` function
- `backend/src/campaign/manager.ts` exports `loadIpContext` function
- `npm --prefix backend run typecheck` passes
</acceptance_criteria>

### Task 5: Frontend passes campaignId to suggest-seeds

<read_first>
- frontend/components/title/use-new-campaign-wizard.ts
- frontend/lib/api.ts (suggestSeeds function)
</read_first>

<action>
1. In `frontend/lib/api.ts`, update `suggestSeeds` to accept optional campaignId:
   ```typescript
   export async function suggestSeeds(premise: string, name?: string, campaignId?: string)
   ```
   Pass campaignId in the request body.

2. In `use-new-campaign-wizard.ts`:
   - The campaign is created BEFORE suggest-seeds is called (campaign load happens first in `handleNextToDna`)
   - Wait, actually — campaign is created in `createCampaignWithSeeds`, not before DNA step. The DNA step happens BEFORE campaign creation.
   - So: campaignId is NOT available during suggest-seeds. The ipContext must be saved AFTER campaign creation.
   - Solution: In `createCampaignWithSeeds`, after campaign is created and loaded, save ipContext to config via a new API call or pass it to the generate endpoint.
   - Simplest: Pass ipContext to the `/api/worldgen/generate` body. The generate route saves it to config and uses it.

3. Update generateWorldSchema in schemas.ts to accept optional ipContext field.
4. Update generate route to save ipContext if provided in request body.
</action>
