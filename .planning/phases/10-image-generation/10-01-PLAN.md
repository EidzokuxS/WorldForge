  <action>
Create `backend/src/images/` module with 4 files:

**generate.ts** — Provider-agnostic image generation:
- `generateImage(opts: { prompt: string; provider: Provider; model: string; size?: string }): Promise<Buffer>`
- Uses plain `fetch()` to call OpenAI-compatible `/v1/images/generations` endpoint (works for OpenAI, OpenRouter, fal, GLM, any compatible API)
- Request body: `{ model, prompt, n: 1, size: size || "1024x1024", response_format: "b64_json" }`
- Auth header: `Authorization: Bearer ${provider.apiKey}`
- Parse response: `response.data[0].b64_json` -> `Buffer.from(b64, "base64")`
- On non-2xx response, throw `AppError` with status and message from response body
- `isImageGenerationEnabled(settings: Settings): boolean` — returns `settings.images.enabled && settings.images.providerId !== NONE_PROVIDER_ID`
- `resolveImageProvider(settings: Settings): { provider: Provider; model: string } | null` — resolves provider from settings.images.providerId against settings.providers, returns null if disabled or not found

**prompt-builder.ts** — Game-state to image prompt:
- `buildPortraitPrompt(opts: { name: string; race: string; gender: string; age: string; appearance: string; tags: string[]; stylePrompt: string }): string`
  - Format: `"Portrait of {name}, {race} {gender}, {age}. {appearance}. Tags: {relevant tags joined}. {stylePrompt}"`
  - Filter tags to appearance-relevant ones (skip wealth/skill/relationship tags)
- `buildLocationPrompt(opts: { locationName: string; tags: string[]; premise: string; stylePrompt: string }): string`
  - Format: `"Fantasy landscape: {locationName}. {tags joined}. World: {premise truncated to 200 chars}. {stylePrompt}"`
- `buildScenePrompt(opts: { eventText: string; locationName: string; premise: string; stylePrompt: string }): string`
  - Format: `"Scene illustration: {eventText}. Setting: {locationName}. World: {premise truncated to 200 chars}. {stylePrompt}"`

**cache.ts** — Disk-based image cache in campaign directory:
- `getImagesDir(campaignId: string, type: "portraits" | "locations" | "scenes"): string` — returns `campaigns/{id}/images/{type}/`
- `ensureImageDir(campaignId: string, type: "portraits" | "locations" | "scenes"): void` — mkdirSync recursive
- `getCachedImage(campaignId: string, type: "portraits" | "locations" | "scenes", filename: string): Buffer | null` — read file or return null
- `cacheImage(campaignId: string, type: "portraits" | "locations" | "scenes", filename: string, data: Buffer): string` — write file, return relative path
- `imageExists(campaignId: string, type: "portraits" | "locations" | "scenes", filename: string): boolean`
- Filename convention: `{entityId}.png` for portraits/locations, `{eventId}.png` for scenes

**index.ts** — Barrel export for all public functions from generate, prompt-builder, cache.

**routes/images.ts** — Image serving route:
- `GET /:campaignId/:type/:filename` — serves cached images
  - Validate `campaignId` with `assertSafeId()`
  - Validate `type` is one of `"portraits" | "locations" | "scenes"` (return 400 otherwise)
  - Validate `filename` matches `/^[\w-]+\.png$/` (return 400 otherwise)
  - Read file from `getImagesDir(campaignId) + "/" + type + "/" + filename`
  - If file not found, return 404
  - Return with `Content-Type: image/png`, `Cache-Control: public, max-age=86400`
- `POST /generate` — on-demand image generation endpoint:
  - Body: `{ campaignId: string, type: "portrait" | "location" | "scene", entityId: string, prompt: string }`
  - Uses `resolveImageProvider(loadSettings())` — return 400 if null (disabled/unconfigured)
  - Calls `generateImage()` with resolved provider
  - Caches result via `cacheImage()`
  - Returns `{ ok: true, path: "/api/images/{campaignId}/{type}/{entityId}.png" }`
  - Add Zod schema `imageGenerateSchema` to schemas.ts

**index.ts** — Register route:
- Import images route: `import images from "./routes/images.js"`
- Mount at `/api/images`: `app.route("/api/images", images)`
