# Plan 52-01 — Preserve full record metadata and render an advanced NPC inspector

## Goal

Expose the full structured character record on World Review NPC cards without changing save semantics.

## Tasks

1. Preserve `characterRecord` through the editable scaffold and character result normalization where needed.
2. Build a read-only advanced inspector component that renders:
   - richer identity
   - grounding
   - power profile
   - sources / continuity
   - runtime/provenance slices
   - raw JSON disclosure
3. Wire the inspector into NPC cards in `NpcsSection`.
4. Add focused frontend tests for:
   - scaffold preservation of `characterRecord`
   - inspector rendering of power-profile data
   - no regression to normal NPC card editing flow

## Watch Items

- Do not let frontend-only inspector metadata break `saveWorldEdits`.
- Prefer effective draft/rendered values over stale shallow fields when possible.
- Keep raw JSON subordinate to the structured inspector.
