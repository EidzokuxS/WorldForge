Based on Task 1 evaluation log, identify issues and apply fixes.

COMMON ISSUE -> FIX MAPPING:

1. Oracle probabilities too generous (everything 70+):
   - Edit ORACLE_SYSTEM_PROMPT in oracle.ts
   - Add: "Do NOT default to high chances. A character without explicit skill tags for an action starts at ~30%. Only specialized training (relevant tags) pushes above 50%."
   - Add more calibration examples at different probability bands

2. Oracle probabilities too harsh (everything under 30):
   - Add: "Characters WITH relevant skill tags should generally succeed. [Master Swordsman] attacking a common guard should be 80+."
   - Ensure the Oracle accounts for tag advantages properly

3. Storyteller narration too generic / not using IP terminology:
   - Edit SYSTEM_RULES in prompt-assembler.ts
   - Add: "Use the terminology, concepts, and naming conventions from the [WORLD PREMISE] and [LORE CONTEXT]. If the world has specific terms for abilities, locations, or social structures, use those terms consistently instead of generic fantasy equivalents."

4. Storyteller not differentiating outcome tiers:
   - Edit OUTCOME_INSTRUCTIONS in turn-processor.ts
   - Make instructions more specific: strong_hit should include sensory detail of mastery, weak_hit should name the specific complication, miss should have concrete consequences (not just "you fail")

5. NPCs passive / not taking autonomous actions:
   - Edit NPC agent system prompt in npc-agent.ts
   - Add: "You SHOULD take at least one action when the player is present. Passing (no tools) should only happen if nothing in your goals or the current situation warrants action."
   - Increase NPC temperature from 0 to 0.3 for more varied behavior

6. Quick actions not contextually relevant:
   - Edit tool schema for offer_quick_actions in tool-schemas.ts
   - Add better description: "Suggest 2-4 actions that a player would naturally consider in this specific situation. Reference present NPCs by name, available items, and current threats."

7. Lore cards not surfacing:
   - Check if embedder is configured and working
   - Verify lore card search is returning results

AFTER APPLYING FIXES:
- Re-test the specific failing scenarios through the browser
- Play 3-5 targeted turns that previously scored poorly
- Verify improvement (quality score should increase by >= 1 point)
- browser_take_screenshot for before/after comparison

If no issues found (average quality >= 4.0): document that Known IP playtest passed clean.
