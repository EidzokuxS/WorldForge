Based on Task 1 extended session findings, apply fixes targeting long-session quality.

LONG-SESSION SPECIFIC ISSUES -> FIXES:

1. World consistency breaks (LLM contradicts premise):
   - Strengthen SYSTEM_RULES in prompt-assembler.ts
   - Add: "CRITICAL: Never contradict the [WORLD PREMISE]. If the premise says 'no sun', NEVER describe sunlight. If the premise defines specific technology levels, respect them."

2. Memory not surfacing relevant past events:
   - Check episodic memory retrieval in prompt-assembler.ts
   - Verify embedder is configured and events are being stored
   - Consider adjusting composite score weights if recency dominates too much

3. Faction ticks not producing visible changes:
   - Check world-engine.ts faction prompt
   - Make faction actions more concrete: "Your faction_action MUST include a specific, observable change — territory gained/lost, resource acquired/depleted, alliance formed/broken. Vague actions like 'continued to plan' are not acceptable."
   - Verify tick interval (default 10) — may need to lower to 5 for more dynamic world

4. NPC reflection not triggering:
   - Check if events are being logged with proper importance scores
   - Consider lowering REFLECTION_THRESHOLD from 15 to 10 for more frequent reflection
   - Verify that reflection results (beliefs, goals) actually appear in NPC prompts

5. Off-screen NPCs doing nothing interesting:
   - Edit npc-offscreen.ts prompt to be more directive
   - Add: "Each NPC update should describe something SPECIFIC they did, not just 'continued pursuing goals.' Name locations, actions, and consequences."

6. Game feels repetitive / not fun:
   - Increase Storyteller temperature slightly (0.7 -> 0.8) for more creative narration
   - Add variety to OUTCOME_INSTRUCTIONS: strong_hit should sometimes include unexpected bonuses, not just "full success"
   - Add directive to offer_quick_actions tool: "Vary suggestions — don't always suggest the same types of actions"

AFTER APPLYING FIXES:
- Re-play 5 turns in the same campaign to verify improvements
- Focus on the specific scenarios that scored poorly
- browser_take_screenshot for re-test results

If no issues found (all scores pass): document clean playtest result.
