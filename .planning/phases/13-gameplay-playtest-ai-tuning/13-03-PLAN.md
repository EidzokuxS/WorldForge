COMBAT-SPECIFIC ISSUES -> FIXES:

1. HP not changing during combat (set_condition not called):
   - Check tool-schemas.ts set_condition definition
   - Ensure description clearly instructs: "Call this tool whenever a character takes damage or is healed. Always specify the new HP value."
   - Add to SYSTEM_RULES: "In combat, you MUST call set_condition when damage is dealt. Never describe damage without updating HP."

2. HP=0 produces automatic "you die" instead of contextual outcome:
   - Strengthen the HP=0 instructions in SYSTEM_RULES (prompt-assembler.ts)
   - Make it even more explicit: "When HP reaches 0, read the SCENE context to determine the outcome type. Non-lethal scenarios (bar brawl, sparring, pit fight): knockout/submission. Lethal scenarios (death match, assassination, monster attack): death is possible but consider dramatic alternatives (capture, last-second rescue, enemy mercy)."

3. Quick actions repetitive in combat:
   - Edit offer_quick_actions tool description
   - Add: "In combat, vary suggestions each turn. Include: specific combat moves referencing available weapons/skills, defensive maneuvers, environmental actions (throw sand, kick over table), social options (surrender, intimidate), and escape routes."

4. Oracle not factoring in HP/wounded state:
   - Check if actor tags include HP-related status
   - Consider adding HP context to Oracle payload (not just tags)
   - Or add a directive to Oracle: "If the actor is wounded (low HP), reduce chance for physically demanding actions by 10-20%."

5. Checkpoint not auto-triggering at HP <= 2:
   - This is a code bug, not a prompt issue. Check the auto-checkpoint logic in the turn pipeline.
   - Verify the onPostTurn callback includes checkpoint logic

FINAL VALIDATION:
- After all fixes, re-play 3 combat turns in the arena campaign
- Verify HP tracks, death narration is contextual, quick actions vary
- browser_take_screenshot for final validation
