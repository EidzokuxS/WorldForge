Create a browser E2E test script using Playwright MCP following the pattern from e2e/19-02-gameplay-browser-e2e.ts. Use the same campaign from API tests (Phase 19/21 campaign).

**Prerequisites:** Backend on :3001, frontend on :3000, campaign with prior chat history from API tests.

**Browser test sequence:**

1. **Chat history persistence on page load** (no new LLM calls):
   - Navigate to game page: http://localhost:3000/game
   - Wait for chat history to load (narrative log populates)
   - Screenshot: `21-02-task1-01-history-loaded.png`
   - Verify: narrative log contains multiple messages (from prior API test turns)
   - Count visible messages — should be > 0 (prior turns are displayed)

2. **Chat history survives reload**:
   - Record message count or last message text
   - Reload the page (page.reload())
   - Wait for chat history to re-load
   - Screenshot: `21-02-task1-02-after-reload.png`
   - Verify: same messages present after reload (count matches, content identical)

3. **Lore panel semantic search** (needs Embedder):
   - Open lore panel (if it's a tab/panel in sidebar)
   - If lore panel has a search input: type a relevant term from the campaign world
   - Screenshot: `21-02-task1-03-lore-search.png`
   - Verify: search results appear with term+definition
   - If no lore UI search: verify lore cards are displayed in the panel grouped by category

4. **Multi-turn gameplay with context awareness** (needs GLM):
   - Play 2 turns via ActionBar with 60s delay between turns:
     - Turn 1: "I examine the area around me and describe what I see"
     - Turn 2: Reference something from turn 1 context: "I investigate the most interesting thing I noticed"
   - Screenshot after each turn: `21-02-task1-04-turn1.png`, `21-02-task1-05-turn2.png`
   - Verify turn 2 narrative references or builds on turn 1 context (not a fresh start)
   - This confirms prompt assembly feeds accumulated context to Storyteller

5. **Final state verification**:
   - Screenshot full game page: `21-02-task1-06-final.png`
   - Verify: narrative log shows all messages (original history + new turns)
   - Verify: sidebar shows current world state (location, NPCs if present)

**Scoring:** Each step scores pass/fail. Quality = (passed / 5) * 5.0. Minimum: 4.0/5.0.

**Rate limit handling:** 60s delay between gameplay turns. 300s timeout per turn. Page reload recovery for stuck SSE streams (same as Phase 19-02 pattern).

**Output:** Screenshots in e2e/screenshots/, results JSON at e2e/screenshots/21-02-results.json.
