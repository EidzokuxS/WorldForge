**Bug 5 Fix — Auto-checkpoint after HP drops to 2 or below during a turn:**

The current auto-checkpoint in chat.ts (lines 409-431) runs BEFORE the turn. This means if HP starts at 3 and drops to 2 during the turn, no checkpoint exists. The fix adds a POST-TURN checkpoint trigger.

1. In `turn-processor.ts`, add a new TurnEvent type `"auto_checkpoint"` to the TurnEvent type union:
   ```typescript
   export interface TurnEvent {
     type:
       | "oracle_result"
       | "narrative"
       | "state_update"
       | "quick_actions"
       | "auto_checkpoint"
       | "done"
       | "error";
     data: unknown;
   }
   ```

2. In `processTurn`, after the fullStream loop ends and before yielding "done", check if any `set_condition` tool call resulted in HP <= 2. Use the `toolCallResults` array:
   ```typescript
   // Check if any set_condition dropped HP to danger zone (2 or below)
   const hpDropped = toolCallResults.some((tc) => {
     if (tc.tool !== "set_condition") return false;
     const output = tc.result as Record<string, unknown> | undefined;
     const inner = output?.result as Record<string, unknown> | undefined;
     const newHp = inner?.newHp as number | undefined;
     return newHp !== undefined && newHp <= 2 && newHp > 0; // >0 because HP=0 is game over, not checkpoint
   });

   if (hpDropped) {
     yield { type: "auto_checkpoint", data: { reason: "HP dropped to danger zone" } };
   }
   ```

3. In `chat.ts`, in the `/action` SSE loop, handle the `auto_checkpoint` event. When received, create a checkpoint:
   ```typescript
   if (event.type === "auto_checkpoint") {
     // Fire-and-forget checkpoint creation
     void (async () => {
       try {
         await createCheckpoint(activeCampaign.id, {
           name: "auto-danger",
           description: `Auto-save: HP dropped to danger zone`,
           auto: true,
         });
         await pruneAutoCheckpoints(activeCampaign.id, 3);
       } catch (err) {
         log.warn("Reactive auto-checkpoint failed (non-blocking)", err);
       }
     })();
   }
   ```

   Also forward the event to the client so the UI can show a notification:
   ```typescript
   await stream.writeSSE({
     event: event.type,
     data: JSON.stringify(event.data),
   });
   ```

4. The existing pre-turn auto-checkpoint in chat.ts (lines 409-431) stays as-is — it catches cases where HP was already low at turn start.
