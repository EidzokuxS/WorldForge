| Boundary | Description |
|----------|-------------|
| Backend response to browser state | `_researchArtifact` leaves backend and must survive client state transitions. |
| Browser to worldgen routes | Client request body can omit or include artifact authority. |
| Backend scaffold NPC to review UI | Canonical identity can be lost during draft fallback conversion. |
| Worldgen to generic character ingestion | Adjacent non-worldgen import flows can be mistaken for artifact-backed worldgen paths. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-72-04-02 | Spoofing | client-submitted artifacts | mitigate | Frontend only preserves backend-returned artifact; backend still parses/caps artifacts at route boundary. |
| T-72-04-03 | Tampering | `scaffoldNpcToDraft` identity fallback | mitigate | Tests assert known-IP identity round-trips and fallback-original stays limited to missing draft/new NPC. |
