# 82-02 Plan - Anchored Ephemeral Sublocation Lifecycle

## Goal

Make dynamic local sublocations a real gameplay primitive instead of a half-wired `reveal_location` side effect.

## Requirements Covered

P82-R1, P82-R2, P82-R3, P82-R6, P82-R8.

## Tasks

1. Run GitNexus impact before editing `handleRevealLocation`, location graph traversal, scene presence, or lifecycle helpers.
2. Extend the dynamic location tool contract with local anchor/current-scene semantics and bounded lifetime policy.
3. Ensure created ephemeral scenes store correct parent/anchor/current-scene metadata and return authoritative refs in tool observations.
4. Add cleanup/archive behavior after successful tool-step settlement, narrator packet/final narration, and post-turn simulation, before the next GM Read assembles.
5. Preserve anchored durable event spillover for archived ephemeral scenes.
6. Presence-gate cleanup so active player/focal/key-participant scenes are never archived out from under play.
7. Add regression coverage for active, expired, archived, presence-gated, and remote-rejected ephemeral scenes.

## Verification

- Focused backend tests for tool executor, location graph, location events, and currentScene pass.
- Backend typecheck passes.
- GitNexus detect_changes scope matches the planned runtime seams.

## Completion Notes

The GM should be able to create a booth under a station, a side office under a guild hall, or a temporary alley scene under a district. It should not create an unrelated remote location from local play.
