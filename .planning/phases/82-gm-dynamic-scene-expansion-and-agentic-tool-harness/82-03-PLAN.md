# 82-03 Plan - Support NPC Lifecycle And Scene Placement

## Goal

Make support NPCs useful for play without polluting the world forever or breaking broad/current-scene presence.

## Requirements Covered

P82-R1, P82-R4, P82-R5, P82-R6, P82-R8.

## Tasks

1. Run GitNexus impact before editing `handleSpawnNpc`, `resolveScenePresence`, or any character/NPC placement helpers.
2. Add or refine support NPC tool fields for `locationRef`, role, reason, lifetime, and optional promotion eligibility.
3. Fix broad/current-scene placement for macro, persistent sublocation, and ephemeral scene targets.
4. Add deterministic retire/cleanup behavior for non-promoted temporary/support NPCs.
5. Add explicit promotion path or reuse an existing tier/persistence update path when a support NPC becomes important: remove cleanup eligibility, record promoting turn/provenance, and run or schedule persistent-NPC enrichment.
6. Presence-gate support NPC cleanup so player/focal/key-participant actors are not retired mid-scene.
7. Add regression tests proving sibling sublocation exclusion, cleanup, and promotion.

## Verification

- Focused backend tests for `spawn_npc`, scene presence, `/world.currentScene`, and narrator packet pass.
- Backend typecheck passes.
- No canonical/key NPC can be retired by support NPC cleanup.

## Completion Notes

Support NPCs should feel like a human GM adding a guard, clerk, or witness because the scene needs one. They are not a cheap way to mint permanent cast every turn.
