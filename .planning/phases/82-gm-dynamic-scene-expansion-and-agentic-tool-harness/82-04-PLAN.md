# 82-04 Plan - Agentic GM Tool Harness Observations And Budgets

## Goal

Strengthen the Phase 81 sequential tool loop so the GM gets useful observations after each tool step and the runtime prevents loops/spam without duration caps.

## Requirements Covered

P82-R1, P82-R6, P82-R7, P82-R8.

## Tasks

1. Run GitNexus impact before editing `executeGmToolSteps`, tool schema contracts, or turn processor progress events.
2. Add tool metadata needed by the GM harness: read/write, mutating, idempotent, destructive, local-only, lifecycle-affecting.
3. Normalize tool-step observations into compact model-facing results with refs, state version/diff summaries, warnings, and next allowed tools.
4. Add semantic budgets: max dynamic spawns per turn, max revisions, max unresolved lifecycle cleanup attempts, and max equivalent repeated tool calls using `toolName + normalized anchor/current-scene ref + normalized role/name/kind + lifetime category`.
5. Ensure cross-turn reuse is allowed, rollback resets budgets, and one validation-failure revision path remains allowed.
6. Emit exact SSE/UI stages: `creating-local-scene`, `spawning-support-npc`, `settling-tool-observation`, and `cleaning-transient-scene`.
7. Update GM Action Checklist, GM tool-step, tool-schema descriptions, and prompt-contract tests with reuse/create/retire/promote rules.
8. Add tests for repeated equivalent calls and no duration-based timeout behavior.
