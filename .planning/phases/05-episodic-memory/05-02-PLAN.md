<objective>
Add smart context compression, episodic memory retrieval, and multi-hop graph queries to the prompt assembler.

Purpose: The current prompt assembler takes the last N messages by token count (naive tail-slicing) and has no episodic memory section. Over long sessions this loses important context. This plan adds: (1) smart compression that keeps first messages + last N + anomalous events, (2) episodic memory section from composite-scored vector search, (3) multi-hop relationship graph enrichment for NPCs in scene.

Output: Updated prompt-assembler.ts with 3 new capabilities, new graph-queries.ts helper, compression tests.
</objective>

    **2. Create backend/src/engine/graph-queries.ts:**
    ```typescript
    export interface GraphNode {
      entityId: string;
      entityName: string;
      relationships: Array<{ targetId: string; targetName: string; tags: string[]; reason: string | null }>;
    }

    export function getRelationshipGraph(
      campaignId: string,
      entityIds: string[],
      maxDepth: number = 2
    ): GraphNode[]
    ```
    - Query `relationships` table for campaignId
    - BFS from seed entityIds up to maxDepth hops
    - Resolve entity names by querying players, npcs, locations, factions tables
    - Return array of GraphNode with their direct relationships
    - Pure SQL/Drizzle, no LLM calls

    **3. Update prompt-assembler.ts:**

    a) **Replace `buildConversationSection` with smart compression:**
    - New function `compressConversation(history: ChatMessage[], budgetTokens: number, importanceThreshold: number = 7): PromptSection | null`
    - Always keep: first 2 messages (world setup / character intro)
    - Always keep: last N messages that fit in 60% of conversation budget
    - From remaining middle messages: keep those whose content contains combat/death/discovery markers OR are explicitly tagged with importance (check for `[IMPORTANT]` prefix or keywords like "attack", "killed", "discovered", "betrayed", "died")
    - Drop mundane middle messages first when over budget
    - Insert `[... {N} earlier turns omitted ...]` marker where messages were dropped

    b) **Add `buildEpisodicMemorySection`:**
    - If embedderResult is available and playerAction exists, embed playerAction, then call `searchEpisodicEvents(queryVector, currentTick, 5)`
    - Format each event as: `[Tick {tick}] {text} (importance: {importance})`
    - Return as PromptSection with name "EPISODIC MEMORY", priority 5, canTruncate true

    c) **Enrich NPC states with relationship graph:**
    - In `buildNpcStatesSection`, after building NPC blocks, call `getRelationshipGraph(campaignId, npcIds, 2)`
    - For each NPC, append their relationship connections as indented lines:
      `  Relationships: {tags.join(", ")} with {targetName}`
    - This replaces the separate `buildRelationshipsSection` — fold relationship data directly into NPC blocks for richer context
    - Keep `buildRelationshipsSection` as fallback for player relationships not covered by NPC graph

    d) **Wire new sections into `assemblePrompt`:**
    - Add episodic memory section between lore context and conversation
    - Pass `currentTick` from campaign config to episodic search
    - Use `compressConversation` instead of old `buildConversationSection`
