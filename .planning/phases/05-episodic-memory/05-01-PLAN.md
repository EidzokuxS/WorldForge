       - Calls `embedTexts([text], provider)` to get real vector
       - Opens the `episodic_events` LanceDB table
       - Updates the row matching `eventId` — LanceDB doesn't have UPDATE, so: query all rows, find matching ID, update vector in-memory, drop table and recreate with updated rows. (Follow the pattern — LanceDB is append-only, but tables are small per campaign.)

       - For each candidate, compute composite score:
         - `similarity = 1 - cosineDistance` (LanceDB returns `_distance`)
         - `recency = currentTick > 0 ? candidate.tick / currentTick : 1.0`
         - `importanceNorm = candidate.importance / 10`
         - `composite = similarity * 0.4 + recency * 0.3 + importanceNorm * 0.3`
       - Sort by composite descending, return top `limit` (default 5)

    ```typescript
    onPostTurn: async (summary) => {
      // Find log_event tool calls from this turn
      const logEvents = summary.toolCalls.filter(tc => tc.tool === "log_event");
      if (logEvents.length === 0) return;

      // Resolve embedder — if not available, skip silently
      const emb = resolveEmbedder(settings);
      if ("error" in emb) return;

      // For each log_event call, embed the stored event asynchronously
      for (const logEvent of logEvents) {
        const result = logEvent.result as Record<string, unknown> | undefined;
        const eventId = result?.eventId as string | undefined;
        const text = (logEvent.args as Record<string, unknown>)?.text as string | undefined;
        if (!eventId || !text) continue;

        try {
          await embedAndUpdateEvent(eventId, text, emb.resolved.provider);
        } catch (err) {
          log.warn("Failed to embed episodic event", err);
        }
      }
    }
    ```
