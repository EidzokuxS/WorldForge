    **1. Create `backend/src/engine/token-budget.ts`:**

    Export `estimateTokens(text: string): number` -- returns `Math.ceil(text.length / 4)`.

    Export `PromptSection` interface:
    ```typescript
    interface PromptSection {
      name: string;
      priority: number;  // 0 = highest (never cut)
      content: string;
      estimatedTokens: number;
      canTruncate: boolean;
    }
    ```

    Export `DEFAULT_BUDGETS` object with percentage allocations:
    - systemRules: 0.05, worldPremise: 0.03, scene: 0.05, playerState: 0.03
    - npcStates: 0.10, actionResult: 0.03, loreContext: 0.08, recentConversation: 0.25
    - responseHeadroom: 0.25 (reserved, not a section)

    Export `allocateBudgets(contextWindow: number): Record<string, number>` -- multiplies each budget percentage by contextWindow to get absolute token limits per section.

    Export `truncateToFit(sections: PromptSection[], maxTokens: number): PromptSection[]` -- if total exceeds maxTokens, iteratively trim content from highest-priority-number (lowest importance) sections that have `canTruncate=true`. For conversation: trim oldest messages first. For lore/NPC: reduce count. Return new array with trimmed content and updated estimatedTokens. Never touch canTruncate=false sections.

    **2. Create `backend/src/engine/prompt-assembler.ts`:**

    Export `AssembledPrompt` interface:
    ```typescript
    interface AssembledPrompt {
      sections: PromptSection[];
      totalTokens: number;
      budgetUsed: number;  // percentage 0-100
      formatted: string;   // final prompt string
    }
    ```

    Export `AssembleOptions` interface:
    ```typescript
    interface AssembleOptions {
      campaignId: string;
      contextWindow: number;  // model's context window in tokens
      actionResult?: {         // Oracle result (optional, not available for Oracle call itself)
        chance: number;
        roll: number;
        outcome: string;
        reasoning: string;
      };
      embedderResult?: ResolveResult;  // for lore vector search
      playerAction?: string;           // current action text, used as lore search query
    }
    ```

    Export `async function assemblePrompt(options: AssembleOptions): Promise<AssembledPrompt>`.

    Implementation:
    - Read campaign config via `readCampaignConfig(campaignId)` for premise
    - Query `players` table for player state (handle null gracefully -- omit section)
    - If player has `currentLocationId`, query `locations` + `npcs` + `items` at that location for scene context
    - Query `npcs` at player's location for NPC states (include persona, tags, goals, beliefs)
    - Query `relationships` involving player or NPCs in scene
    - If `embedderResult` is resolved and `playerAction` provided: embed playerAction text via `embedTexts()`, then `searchLoreCards(queryVector, 3)` for top 3 lore cards. If embedder not configured or fails, skip lore section gracefully with a debug log.
    - Read chat history via `getChatHistory(campaignId)`, take last N messages that fit budget
    - Format system rules section with hardcoded game rules template (narrator role, tag-based system, HP 1-5, no hallucinating items)
    - If `actionResult` provided, format as [ACTION RESULT] block
    - Build PromptSection array with priorities: systemRules=0, worldPremise=1, scene=2, playerState=3, npcStates=4 (canTruncate), actionResult=5, loreContext=6 (canTruncate), recentConversation=7 (canTruncate)
    - Call `truncateToFit(sections, contextWindow - responseHeadroom)`
    - Format output: join sections as `[SECTION NAME]\n{content}` separated by `\n\n`
