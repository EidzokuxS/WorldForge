  <action>
Create `backend/src/worldgen/worldbook-importer.ts` with three exported functions:

1. **`parseWorldBook(json: unknown): WorldBookEntry[]`** — Parse raw SillyTavern WorldBook JSON. WorldBook format is `{ entries: Record<string, WorldBookEntry> }` where each entry has `key` (string[]), `keysecondary` (string[]), `comment` (name), `content` (lore text), `selective`, `constant`, `vectorized`, `selectiveLogic`, `excludeRecursion`, `position`, `depth`, `extensions`, etc. Extract only `comment` (as `name`) and `content` (as `text`). Strip HTML tags from content using regex `/<[^>]+>/g`. Deduplicate by name (case-insensitive, keep first). Return array of `{ name: string; text: string }`.

Define Zod schema for input validation: `worldBookJsonSchema = z.object({ entries: z.record(z.object({ comment: z.string(), content: z.string() }).passthrough()) }).passthrough()` — passthrough allows extra SillyTavern fields without failing validation.

2. **`classifyEntries(entries: WorldBookEntry[], role: ResolvedRole): Promise<ClassifiedEntry[]>`** — Send all entries to Generator LLM in a single `generateObject` call. Schema: array of `{ name: string, type: "character" | "location" | "faction" | "bestiary" | "lore_general", summary: string }`. The prompt should instruct the LLM to classify each entry based on its content, with these guidelines:
   - character: describes a person, creature with personality, NPC
   - location: describes a place, area, region
   - faction: describes an organization, group, guild, army
   - bestiary: describes a creature type, monster species (no individual personality)
   - lore_general: world rules, history, magic systems, concepts, items

3. **`importClassifiedEntries(campaignId: string, entries: ClassifiedEntry[], embedderResult: ResolveResult): Promise<ImportResult>`** — Route each entry by type:
   - `character` entries: insert into `npcs` table with `tier: "key"`, `persona` from text, tags inferred as empty array, goals as `{ short_term: [], long_term: [] }`, `currentLocationId: null`, `factionName: null`. Use `crypto.randomUUID()` for IDs, pattern from scaffold-saver.ts.
   - `location` entries: insert into `locations` table with description from text, `tags: []`, `isStarting: false`, `connectedTo: "[]"`.
   - `faction` entries: insert into `factions` table with `tags: []`, `goals: [summary]`, `assets: []`.
   - `bestiary` and `lore_general` entries: collect as lore cards `{ term: name, definition: text, category: "concept" }` (bestiary gets category "npc"), then call `storeLoreCards(cards, embedderResult)`.
   - Return `{ imported: { characters: number, locations: number, factions: number, loreCards: number } }`.

Use Drizzle ORM (import from `../db/index.js` and `../db/schema.js`), wrap all DB writes in a transaction. Use `createLogger("worldbook-importer")` for logging.

Add two API endpoints to `backend/src/routes/worldgen.ts`:

**`POST /parse-worldbook`** — accepts `{ campaignId: string, worldbook: object }`. Calls `parseWorldBook` then `classifyEntries`. Returns `{ entries: ClassifiedEntry[] }` for frontend preview. Requires active campaign + Generator role.

**`POST /import-worldbook`** — accepts `{ campaignId: string, entries: ClassifiedEntry[] }`. Calls `importClassifiedEntries`. Returns `{ imported: { characters, locations, factions, loreCards } }`. Requires active campaign + Embedder role (for lore cards).

Add Zod schemas `parseWorldBookSchema` and `importWorldBookSchema` to `backend/src/routes/schemas.ts`.
