# Phase 74 Structured Prompt Contract Audit

**Created:** 2026-04-28
**Purpose:** Convert the Phase 73 structured-output inventory into an audit of what the model is actually told before it returns structured data.

## Method

Commands used from repo root:

- `rg -n "safeGenerateObject\(|generateObject\(|streamObject\(|Output\.json|response_format|json_schema|json_object" backend/src shared frontend`
- `rg -n "import \{[^}]*generateObject|from [\"']ai[\"']|from \"ai\"" backend/src --glob "!**/__tests__/**"`
- `rg -n "safeGenerateObject\(" backend/src --glob "!**/__tests__/**"`
- `rg -n "generateObject\(" backend/src --glob "!**/__tests__/**"`

## Current Boundary Shape

- Production direct AI SDK `generateObject` imports are blocked by `backend/src/ai/__tests__/structured-output-boundary.test.ts`.
- Many production files intentionally alias `safeGenerateObject` as `generateObject`, so a simple call-site grep still finds many `generateObject(...)` calls.
- Phase 73 fixed provider strategy, JSON mode, repair, trace labels, and representative conformance. It did not prove every prompt has a clear schema/tool contract.

## Known Prompt-Contract Gaps

| Surface | Current model-facing wording | Gap |
|---|---|---|
| `buildGeneratedContextPrompt` | "Compile generatedContext under the source usage rules." | Does not show the `generatedContext` object shape, especially `citations` object array and `canonicalNames` object. Directly matches observed failures. |
| `buildWorldBrainPrompt` | "Return only the bounded structured scene-direction object." | No visible field list, nested shape, caps, or example for the loose/strict world-brain schemas. |
| `buildJudgeAdjudicationContract` | "Decide backend actions as structured data only." | Needs the action object shape and nested runtime tool input contracts, not only general intent. |
| `buildDefaultScenePlannerPrompt` | Lists semantic fields and allowed tools. | Still underspecifies nested `plannedActions[].input` per tool, especially optional quick actions. |
| `character/generator.ts` and `character/npc-generator.ts` | Lists flat field names. | Needs audit for rich nested fields, caps, empty-list/null behavior, and compact examples. |
| `worldgen/scaffold-steps/*` | Mixed per-step natural-language prompts. | Needs consistent schema contract snippets for plan/detail/repair outputs. |
| `backfill-personality.ts` | Builds an operational personality-generation prompt around `safeGenerateObject`. | Production-supported script seam needs the same marker/shape/no-invention contract as runtime character prompts, or an explicit documented exclusion. Phase 74 chooses inclusion in Plan 74-06. |

## Repair Policy To Preserve

Backend may:

- Generate backend-owned IDs.
- Resolve explicit actor refs from allowed SceneFrame actors.
- Trim strings and cap arrays.
- Normalize known aliases such as `payload` -> `input` when the meaning is already explicit.
- Drop optional non-executable UI-only outputs when no recoverable object exists.

Backend must not:

- Invent missing semantic action text, actor intent, world lore, source roles, power facts, canonical names, or target meaning.
- Treat repair success as proof that the original prompt is stable.
- Claim a provider/model is stable for a case if it only passes after fallback or repair.
