<objective>
Backend endpoints for retry, undo, and edit — plus the state snapshot system that makes undo/retry safe by capturing pre-turn game state and restoring it on rollback.

Purpose: Without state snapshots, retry/undo would leave orphaned entities, wrong HP, and stale tags. The snapshot system ensures mechanical consistency when rolling back narrative.

Output: Three new POST endpoints (`/api/chat/retry`, `/api/chat/undo`, `/api/chat/edit`) and a state snapshot module.
</objective>

    **1. Create `backend/src/engine/state-snapshot.ts`:**

    Export `TurnSnapshot` interface:
    ```typescript
    export interface TurnSnapshot {
      playerHp: number;
      playerTags: string;
      playerLocationId: string | null;
      playerEquippedItems: string;
      tick: number;
      // IDs of entities created during the turn (for deletion on rollback)
      spawnedNpcIds: string[];
      spawnedItemIds: string[];
      revealedLocationIds: string[];
      createdRelationshipIds: string[];
      createdChronicleIds: string[];
    }
    ```

    Export `captureSnapshot(campaignId: string): TurnSnapshot` — reads player row from DB (hp, tags, currentLocationId, equippedItems) and currentTick from campaign config. All spawned* arrays start empty.

    Export `restoreSnapshot(campaignId: string, snapshot: TurnSnapshot): void` — restores player state (hp, tags, location, equipped) from snapshot, decrements tick back to snapshot.tick via writing to config.json, and deletes any entities in the spawned* arrays:
    - DELETE FROM npcs WHERE id IN spawnedNpcIds
    - DELETE FROM items WHERE id IN spawnedItemIds
    - DELETE FROM locations WHERE id IN revealedLocationIds (also remove their IDs from connectedTo of adjacent locations)
    - DELETE FROM relationships WHERE id IN createdRelationshipIds
    - DELETE FROM chronicle WHERE id IN createdChronicleIds

    For revealed location cleanup: before deleting, read the location's connectedTo to find adjacent locations, then update each adjacent location's connectedTo array to remove the deleted location's ID.

    **2. Update `backend/src/engine/index.ts`:** Re-export `captureSnapshot`, `restoreSnapshot`, `TurnSnapshot` from state-snapshot.ts.

    **3. Add to `backend/src/campaign/chat-history.ts`:**

    `popLastMessages(campaignId: string, count: number): ChatMessage[]` — reads chat history, removes last `count` messages from the array, writes back, returns the removed messages.

    `replaceChatMessage(campaignId: string, index: number, newContent: string): boolean` — reads chat history, validates index is in range and message is role=assistant, replaces content, writes back. Returns true on success, false if index invalid or not assistant message.

    `getLastPlayerAction(campaignId: string): string | null` — reads chat history from end, finds the last message with role=user, returns its content. Returns null if no user messages.
