POST /api/chat/retry
- No body required
- Returns SSE stream (same format as /api/chat/action)
- Events: oracle_result, narrative, state_update, quick_actions, done, error

POST /api/chat/undo
- No body required
- Returns JSON: { success: true, messagesRemoved: 2 }
- Returns 400 if nothing to undo

POST /api/chat/edit
- Body: { messageIndex: number, newContent: string }
- Returns JSON: { success: true }
- Returns 400 if invalid index or not assistant message

    **1. Add API helpers to `frontend/lib/api.ts`:**

    `chatRetry(): Promise<Response>` — calls `apiStreamPost("/api/chat/retry", {})`. Returns raw Response for SSE parsing.

    `chatUndo(): Promise<{ success: boolean; messagesRemoved: number }>` — calls `apiPost("/api/chat/undo", {})`.

    `chatEdit(messageIndex: number, newContent: string): Promise<{ success: boolean }>` — calls `apiPost("/api/chat/edit", { messageIndex, newContent })`.

    If `apiPost` doesn't exist yet, add it following the same pattern as `apiGet` but with POST method and JSON body.

    **2. Add retry/undo handlers to `frontend/app/game/page.tsx`:**

    `handleRetry` async function:
    - Guard: if isStreaming, return
    - Set isStreaming=true, clear quickActions, clear lastOracleResult
    - Remove last assistant message from local messages state (optimistic UI)
    - Add empty assistant placeholder: `{ role: "assistant", content: "" }`
    - Call `chatRetry()` to get SSE stream
    - Parse with `parseTurnSSE` using same handlers as submitAction (onNarrative accumulates text, onOracleResult sets oracle, onQuickActions sets actions, onDone refreshes world data)
    - On error: remove empty placeholder, show toast
    - Finally: set isStreaming=false

    `handleUndo` async function:
    - Guard: if isStreaming, return
    - Call `chatUndo()`
    - On success: remove last 2 messages from local messages state, clear quickActions, clear lastOracleResult, refresh world data, show toast "Last action undone"
    - On error: show toast with error

    `handleEdit` async function (messageIndex: number, newContent: string):
    - Call `chatEdit(messageIndex, newContent)`
    - On success: update local messages state at that index
    - On error: show toast

    Pass handleRetry, handleUndo to NarrativeLog as props. Pass handleEdit as `onEdit` prop.

    Also derive `canRetryUndo` — true when: not streaming AND messages.length >= 2 AND last message is assistant.

    **3. Update `NarrativeLog` component in `frontend/components/game/narrative-log.tsx`:**

    Add props: `onRetry?: () => void`, `onUndo?: () => void`, `onEdit?: (index: number, content: string) => void`, `canRetryUndo?: boolean`.

    For the LAST assistant message (when canRetryUndo is true and not streaming): render a small button row below the message with:
    - Retry button (RotateCcw icon from lucide-react, tooltip "Retry") — calls onRetry
    - Undo button (Undo2 icon from lucide-react, tooltip "Undo") — calls onUndo

    Style: small muted buttons, `opacity-0 group-hover:opacity-100` transition so they appear on hover. Use `group` class on the message container.

    For ALL assistant messages: make them click-to-edit:
    - Add local state: `editingIndex: number | null`, `editText: string`
    - On click of an assistant message paragraph: set editingIndex to that message's index, set editText to message content
    - When editing: render a `<textarea>` instead of `<p>`, pre-filled with content, with:
      - Auto-resize to fit content (use rows or min-height)
      - Save button (Check icon) — calls onEdit(editingIndex, editText), clears editing state
      - Cancel button (X icon) — clears editing state
      - Escape key cancels, Ctrl+Enter saves
    - After save, show a subtle "(edited)" label next to the message (track edited indices in local state or add an `edited` field)

    Style the textarea to match the narrative font (font-serif, same text size), with a subtle border to indicate edit mode.
