  <behavior>
    - Movement: When playerAction matches movement pattern (starts with "go to", "travel to", "move to", "head to", or similar), the turn processor extracts destination name, resolves it against locations table, checks if destination is in current location's connectedTo array, updates player.currentLocationId, yields state_update event with {type: "location_change", locationId, locationName}.
    - If destination not connected: yield state_update with error explaining which locations are reachable, then continue turn normally (Oracle still evaluates the action — maybe dangerous travel).
    - World data: GET /api/campaigns/:id/world returns items array alongside locations/npcs/factions.
  </behavior>
  <action>
    1. **Add movement handling to turn-processor.ts**: Before calling Oracle in processTurn, detect if playerAction is a movement command. Create helper `detectMovement(action: string): string | null` that returns destination name if action matches movement patterns (case-insensitive regex: `/^(?:go\s+to|travel\s+to|move\s+to|head\s+to|walk\s+to|run\s+to|go)\s+(.+)$/i`). If movement detected:
       - Resolve destination location by name (case-insensitive LOWER() match in locations table, scoped to campaignId)
       - Get player's current location, parse its connectedTo JSON array
       - If destination.id is in connectedTo: update player.currentLocationId to destination.id, yield `{ type: "state_update", data: { type: "location_change", locationId: destination.id, locationName: destination.name } }`
       - If destination NOT in connectedTo but exists: DON'T block — still run through Oracle (the action is "trying to travel to unreachable place"), but add context to sceneContext about available paths
       - If destination not found at all: DON'T block — still run through Oracle (Storyteller might use reveal_location to create it)
       - After successful movement, the rest of the turn runs with the NEW location context

    2. **Enrich world data endpoint**: In backend/src/routes/campaigns.ts, update the GET /api/campaigns/:id/world handler to also query and return items. Query all items for campaignId. Return as `items: Array<{ id, name, tags, ownerId, locationId }>` in the response object.

    3. **Add tests** to turn-processor.test.ts for movement detection helper and movement state_update events.
  </action>
