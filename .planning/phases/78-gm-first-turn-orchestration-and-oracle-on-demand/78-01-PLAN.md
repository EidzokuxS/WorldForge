## Tasks

### Task 1: RED Backend Authority-Boundary Tests

**RED**

- Add tests proving `buildSceneFrame` does not resolve `oracleContext`, combat envelope, target, hostility, or action category from raw text alone.
- Add `gm-turn-decision` tests that define the required GM/Judge paths: `direct`, `roll_oracle`, `tool_plan`, `combat_transition`, `clarification`, `continue`.
- Add turn-processor tests with mocked GM decisions proving:
  - pure speech does not call Oracle;
  - `Continue scene.` does not call Oracle;
  - requested roll calls Oracle once and emits `oracle_result`;
  - raw `"I hit Iru"` alone does not create pre-GM combat authority.

**GREEN**

- Do not make production changes in this plan unless required to compile the tests around importable types.
- If type scaffolding is unavoidable, create only the minimum `backend/src/engine/gm-turn-decision.ts` surface:
  - `gmTurnDecisionSchema`;
  - `type GmTurnDecision`;
  - `runGmTurnDecision(args)` throwing `not implemented`.
- Do not put behavioral prompt/LLM logic into this plan; 78-03 owns behavior.

**VERIFY**

```bash
npm --prefix backend run test -- src/engine/__tests__/scene-frame.test.ts src/engine/__tests__/gm-turn-decision.test.ts src/engine/__tests__/turn-processor.scene-plan.test.ts
```

Expected result after RED: tests fail for missing behavior or missing module, not syntax errors.

### Task 2: RED Route Rollback And Compatibility Tests

**RED**

- Extend `/api/chat/action` route tests so invalid GM-selected tools/IDs restore the pre-turn snapshot and do not persist partial changes.
- Add route assertions that `intent` and `method` are accepted only as compatibility fields; the GM-first path must use raw `playerAction` and neutral packet/decision outputs.
- Add an adversarial compatibility assertion where `intent` contradicts `playerAction`; the engine must ignore `intent`/`method` as product semantics and interpret only the raw player action through GM.
- Assert direct/no-roll turns do not stream `oracle_result`.

**GREEN**

- Keep tests deterministic with mocked judge/storyteller/oracle seams. Do not require live providers.

**VERIFY**

```bash
npm --prefix backend run test -- src/routes/__tests__/chat.scene-plan.test.ts
```

Expected result after RED: failures identify missing GM-first route behavior or current automatic Oracle behavior.

### Task 3: RED Phase 77 `/game` Compatibility Tests

**RED**

- Preserve existing expectations that Send calls `chatAction(campaignId, rawText, rawText, "")`.
- Preserve Continue calling `chatAction(campaignId, "Continue scene.", "Continue scene.", "")`.
- Add a no-roll/direct-turn UI test proving stale Oracle/dice output is not shown when the streamed turn contains no `oracle_result`.
- Assert no required Act/Speak/Observe mode control appears.

**GREEN**

- No UI source changes in this plan unless a test harness import must be adjusted.

**VERIFY**

```bash
npm --prefix frontend run test -- app/game/__tests__/page.test.tsx components/game/play-surface/__tests__/action-dock.test.tsx
```

Expected result after RED: existing Phase 77 tests remain meaningful; new no-roll compatibility test fails only where product behavior is not implemented.
