## Tasks

### Task 1: RED `/game` No-Roll Rendering Compatibility

**RED**

- Keep tests proving raw Send and Continue transport remain:
  - `chatAction(campaignId, rawText, rawText, "")`;
  - `chatAction(campaignId, "Continue scene.", "Continue scene.", "")`.
- Add/keep tests proving:
  - a direct/no-roll backend turn clears or does not display stale Oracle/dice result;
  - clarification backend turn renders the GM prompt in the normal narration/beat lane through existing `narrative` SSE / `parseTurnSSE` / `onNarrative` handling, emits no `oracle_result`, `state_update`, or `quick_actions`, executes no tools, mutates no world state, reveals no stale dice, and does not require a separate command mode;
  - mechanics remain Inspect/debug-hidden unless streamed;
  - no Act/Speak/Observe command-mode UI is required.

**GREEN**

- If stale Oracle state exists, clear it at new turn start or when a completed turn has no `oracle_result`, whichever preserves existing Inspect behavior best.
- Do not change the raw action dock into command modes.
- Do not infer travel/combat/mechanics from submitted text in frontend code.
- Preserve staged beats, `Next`/`Auto`/`Log`, drawer state, and draft persistence.
