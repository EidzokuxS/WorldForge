## Tasks

### Task 1: RED Decision Schema And Prompt Contract

**RED**

- Create tests for a `GmTurnDecision` schema with exactly these path values: `direct`, `roll_oracle`, `tool_plan`, `combat_transition`, `clarification`, `continue`.
- Assert the schema/prompt requires:
  - raw `playerAction` as text to interpret;
  - neutral `SceneFrame` evidence;
  - candidate refs from frame only;
  - allowed tools from `frame.allowedTools`;
  - no required Act/Speak/Observe command mode;
  - no backend-owned invented target/tool semantics.
- Assert unsupported tools, invented actor refs, missing required target for tool/combat paths, and over-cap free text fail closed.
- Assert per-path field requirements:
  - `direct`: `directResolutionNotes` / narration guidance only, no state delta fields;
  - `continue`: continuation guidance only, no invented action category;
  - `clarification`: `clarificationPrompt` required;
  - `roll_oracle`: the only path that may request a new backend roll/Oracle call; `rollRequest` must include actor ref, optional target ref, stakes/question, and concrete evidence refs;
  - `tool_plan`: `plannedTools[]` with allowed tool name, actor ref, concrete target refs when required, and schema-shaped input; no new roll request is allowed on this path;
  - `combat_transition`: concrete actor/target refs plus combat framing/stakes; no new roll request is allowed on this path.
- Assert `roll_oracle` is the only GM decision path that may request backend randomness/Oracle. If a tool/combat outcome needs uncertainty, the GM must choose `roll_oracle` first; the later continuation/tool/combat step may consume the existing roll result but must not request a second roll from its own path.
- Assert GM output cannot author time/stat/inventory/location/relationship/HP/condition deltas outside validated tool proposals.

**GREEN**

- Add `backend/src/engine/gm-turn-decision.ts` exporting:
  - `gmTurnDecisionSchema`;
  - `type GmTurnDecision`;
  - `runGmTurnDecision(args)` using `safeGenerateObject`, role `"judge"`, temperature `0`, one repair attempt, and structured-output trace logging consistent with `runScenePlanner`.
- Add prompt-contract helper text that explicitly states backend owns IDs, validation, random rolls, deterministic math, persistence, rollback, and final truth.
- Include compact examples for each allowed path, but do not include Marinara-specific architecture or UI language.
- Make missing required per-path fields fail closed instead of asking backend code to infer them from raw prose.

**VERIFY**

```bash
npm --prefix backend run test -- src/engine/__tests__/gm-turn-decision.test.ts
npm --prefix backend run typecheck
```

### Task 2: Integrate Decision Output With Existing ScenePlan Contracts

**RED**

- Add tests proving `runScenePlanner` can consume a GM decision and optional Oracle result without requiring Oracle for direct/continue/clarification paths.
- Assert ScenePlanner still maps semantic refs through backend validation and still fails closed on unsupported tools/refs.

**GREEN**

- Extend `runScenePlanner` arguments so `oracleResult` is optional when GM decision path does not request a roll.
- Add decision-aware prompt context that tells ScenePlanner:
  - the GM decision path is binding for this turn;
  - Oracle outcome is present only when requested and then binding;
  - direct/continue/clarification paths may produce no mutating tools;
  - direct/continue/clarification paths produce a zero-action/no-mutation ScenePlan-compatible artifact or equivalent validated artifact for the existing finalization/narration tail;
  - tool/combat paths must use concrete refs/tools from candidates/allowed tools and must not trigger Oracle directly.
- Preserve existing semantic-to-strict mapping, ID generation, repair policy, and one repair attempt.

**VERIFY**

```bash
npm --prefix backend run test -- src/engine/__tests__/gm-turn-decision.test.ts src/engine/__tests__/scene-planner.test.ts
```

## Acceptance Criteria

- GM/Judge is the first authority to interpret raw player text after neutral state retrieval.
- Decision schema covers all required paths and fails closed rather than manufacturing missing target/tool meaning.
- ScenePlanner no longer requires an Oracle result for no-roll paths.

## Rollback Notes

If the new module causes structured-output regressions, revert this plan's production module and prompt changes while keeping RED tests. Do not fold interpretation back into `SceneFrame` or route code.
