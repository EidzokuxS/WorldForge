# 88-07 Plan: Memory, Beliefs, and Knowledge Propagation

## Goal

Build a provenance-rich memory and knowledge layer where truth, belief, reports, rumors, observations, and summaries are separate and rollback-aware.

## Scope

- Add belief/report/rumor records or equivalent knowledge model.
- Add SQLite-backed knowledge tables or equivalent authority-owned persistence for claims, beliefs, reports, rumors, and memory provenance.
- Add memory write policy.
- Add required-before-use reflection where an actor is about to act with stale/high-importance evidence.
- Add hybrid retrieval hooks: structured scope filters, lexical/BM25 text recall, and optional vector recall, all source-backed in ContextBudgetTrace.
- Preserve event log as source truth; summaries are indexes, not truth replacement.

## Acceptance

- False claims create claim events and listener beliefs, not backend truth.
- Reports and rumors propagate with source, latency, reliability/confidence, recipients, and knowledge route.
- Memory writes only for consequential observations, reports, promises, threats, orders, relationship/goal changes, tool outcomes, hidden-info revelations, and meaningful state changes.
- ActorFrame retrieval stays small, source-backed, and actor-known.
- Lexical retrieval catches exact names, promises, locations, item ids, and quoted reports even when vector recall misses them.
