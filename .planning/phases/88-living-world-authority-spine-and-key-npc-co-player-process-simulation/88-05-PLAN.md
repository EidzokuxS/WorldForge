# 88-05 Plan: ActorDecisionPacket and Actor Tool Validation

## Goal

Replace prose-driven NPC ticking with structured actor decisions and backend-validated actor tools that commit only through authority.

## Scope

- Add `ActorDecisionPacket`.
- Add actor brain runner over ActorFrame.
- Expand actor tools where backend state can validate them.
- Route actor tool results through the expanded ToolResult contract.
- Include combat/contested-resource validation for actor actions.
- Define combat as actor intent plus backend mechanical resolution; the model may choose tactics, but cannot invent a successful hit, injury, escape, or kill outside ToolResult.

## Acceptance

- Actor output includes actor id, intent, facts used, selected goal, tool calls, belief/plan updates, next decision trigger, and no-action reason.
- Tools validate location, visibility, target, ownership, resources, authority, travel path, combat bounds, elapsed time, and write scope.
- Failed tools produce explicit failure/partial ToolResult and do not create fake successful no-ops.
- Combat and power-mismatch outcomes are represented as authoritative ToolResults with source events, not narration-only summaries.

## Verification

- `npm --prefix backend run test -- src/engine/__tests__/actor-decision-packet.test.ts src/engine/__tests__/actor-tools.test.ts`
- Negative tests: invalid target, hidden target, unreachable destination, nonexistent item, insufficient authority, stale base version.
- Typecheck.
