# 88-02 Plan: POV and Redaction Contracts

## Goal

Define exact internal and player-facing packets so actor brains see only their own knowledge and narrator prose sees only committed visible truth.

## Scope

- Add `ActorFrame` for key NPCs.
- Add `CommandNodeFrame` for faction decision makers.
- Add or harden `PlayerFacingPacket` as the pre-narrator truth boundary.
- Add `ContextBudgetTrace` and prompt/payload leak audits.
- Extend existing `NarratorPacket` guardrails rather than bypassing them.

## Acceptance

- Every ActorFrame fact carries a source route: self-state, direct observation, report/message, rumor, belief, memory, public record, or local affordance.
- ActorDecisionPacket may cite only fact ids present in the ActorFrame.
- Narrator prompt construction throws or fails closed if hidden truth, private actor rationale, unresolved proposals, or offscreen facts would enter visible prose.
