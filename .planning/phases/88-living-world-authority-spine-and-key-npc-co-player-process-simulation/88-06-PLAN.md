# 88-06 Plan: World-Time Plan Executor and Offscreen Catch-Up

## Goal

Implement deterministic continuation of actor plans and just-in-time offscreen catch-up so offscreen life becomes durable state, not batch summary prose.

## Scope

- Add actor plan executor.
- Support long-running work: travel, training, searching, guarding, repairing, preparing operations, resting.
- Resolve generic due world work touching current or soon-exposed scope before SceneFrame/NarratorPacket assembly; WorldThreads attach to this interface in Wave 6.
- Persist inspectable consequences: events, location changes, witnesses, rumors, memories, inventory/status/location changes.

## Acceptance

- Plans run until target world time, decision point, interruption, failure, or precondition break.
- Offscreen changes can later be discovered through state, memories, reports, or changed locations.
- Player observation never requires retroactively inventing why the world changed.
- Broken preconditions create a failure event, replan wake signal, and knowledge/report route for affected actors or factions when they could plausibly learn it.

- Typecheck.
