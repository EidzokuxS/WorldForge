# 88-11 Plan: Rollback, Checkpoint, and Living-World Verification Gate

## Goal

Close Phase 88 with deterministic, integration, trace, and live Playwright proof that the world behaves as a living world under key NPC, faction, memory, rollback, false-claim, combat, and tourist routes.

## Scope

- Verify retry/undo/checkpoint restore for living-world artifacts built in prior waves.
- Add full deterministic invariant suite.
- Add `e2e/88-living-world-playtest.ts` or equivalent named profiles.
- Add LLM/human judge calibration for prose, agency, causality, and playfeel before using soft ratings as release evidence.
- Produce verification matrix with artifact paths, trace paths, state diffs, packet audits, and route outcomes.

## Acceptance

- Restore reverts world version, world time, simulation queue, proposals, actor processes, world threads, faction operations, memories/beliefs, and narrator/cache artifacts created after the restored boundary.
- Stale async proposals cannot commit after retry/undo/checkpoint restore.
- Deterministic tests cover all critical hard-failure classes.
- Live playtests demonstrate tourist agency, key NPC offscreen agency, follow/shadow, faction report latency, false-claim boundary, combat/power mismatch, rollback, memory stress, hidden-truth leakage, and latency stress.
- Soft findings are supported by calibrated rubric examples and cannot be reduced to keyword scoring; lexical scans are limited to obvious banned-template detection.
- If any wave evidence gate fails, later wave/final live routes are skipped until the failing wave is repaired.
