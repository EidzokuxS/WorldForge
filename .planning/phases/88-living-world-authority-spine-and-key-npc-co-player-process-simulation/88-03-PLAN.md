# 88-03 Plan: Versioned Queue, Proposals, and Turn Boundary

## Goal

Replace detached state-bearing post-turn work with an explicit simulation queue/proposal lifecycle and a truthful `done` boundary.

## Scope

- Split required-before-done work from optional proposal work.
- Move state-bearing offscreen/reflection/faction work out of fire-and-forget post-turn mutation.
- Add proposal commit/rebase validation.
- Add minimal SSE stage and trace events for settlement, proposal, stale-reject, and done boundaries.
- Reconcile docs that describe memory/mechanics so they stop promising old detached direct writes.
- Preserve embeddings/images as non-authoritative auxiliary work only.

## Acceptance

- `done` is emitted only after the current authoritative visible boundary is settled.
- Any post-`done` worker can only create proposals or non-authoritative artifacts unless it commits later through version validation.
- Stale proposals, aborted turns, route failures, retries, and checkpoint restores cannot mutate the current world state.
- Existing `simulateOffscreenNpcs`, `checkAndTriggerReflections`, and `tickFactions` remain available only through proposal adapters until later actor/faction waves reconnect them through authority.
- SSE progress exposes the real stage without adding duration caps or duplicate loaders.
