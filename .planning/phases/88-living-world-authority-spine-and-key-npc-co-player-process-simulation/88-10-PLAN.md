# 88-10 Plan: Latency, Context Budget, and Parallel Execution Observability

## Goal

Prove Phase 88 controls latency and context without hiding work, cutting model output, disabling mechanics, or relying on arbitrary turn-duration caps.

## Scope

- Add `TurnLatencyTrace`.
- Add `ContextBudgetTrace`.
- Extend the minimal trace contracts from Wave 3 and write-scope reservation contract from Wave 4 into full observability.
- Add parallel simulation runner throughput with write-scope conflict handling; correctness is not deferred to this wave.
- Track serialized LLM groups, parallel groups, retries, token usage, prompt/output size, actor waits, narrator waits, and proposal/cache effects.

## Acceptance

- Normal turns target four or fewer serialized LLM decision groups; heavy turns target five or fewer.
- Violations are diagnostics and optimization triggers, not silent skips.
- Independent actor/command/thread jobs run in parallel only when write scopes do not conflict.
- Context budget hard-fails hidden truth in narrator, ActorFrame facts the actor cannot know, full-history prompt dumps, source-free memories, and summary-as-truth.
- Serialized group accounting starts no later than Wave 3; this wave adds aggregate reports, alerts, and live-route proof.
