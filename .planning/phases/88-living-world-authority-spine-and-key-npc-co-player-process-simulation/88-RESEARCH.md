## Target Architecture

The target mental model is "backend kernel, player/key NPCs as user-space processes, tools as syscalls, event log as causal filesystem, memory as hierarchy/cache, scheduler as dispatcher, narrator as renderer of committed visible truth." [VERIFIED: docs/WorldForge_Key_NPC_AI_players_latency_memory_budget_full_report_v3.md:139-155]

```text
SSE /api/chat/action
  -> acquire campaign turn lock
  -> load WorldState(version, world_time, current tick)
  -> resolve required exposed simulation catchup
  -> build SceneFrame from committed truth + player-known facts
  -> GM Read / Oracle on demand / GM tool loop
  -> commit player-visible state deltas + events
  -> wake present KeyActorProcesses
       -> build ActorFrame per actor POV
       -> actor decision packet
       -> actor tools through backend validation
       -> commit local reactions
  -> advance world_time by elapsed duration
  -> process due deterministic plan steps and mandatory exposed jobs
  -> stage optional offscreen proposals with base_version/read_set/write_scope
  -> write observations, reports, rumors, memories, and required reflections
  -> build PlayerFacingPacket/NarratorPacket from visible committed truth only
  -> final narration + grounding guard
  -> done(version, world_time)
```

The architecture must preserve this invariant: after `done`, detached jobs may prepare proposals, embeddings, compaction, or review artifacts, but may not mutate state the next GM Read can observe unless the mutation commits at a later authoritative boundary with version/rebase validation. [VERIFIED: docs/WorldForge_Key_NPC_AI_players_latency_memory_budget_full_report_v3.md:431-435; VERIFIED: docs/WorldForge_Key_NPC_AI_players_latency_memory_budget_full_report_v3.md:721-758]

1. No state-bearing mutation may occur outside an authoritative commit with `base_version`, write scope, and postcondition validation. 3. Player-facing narrator must never receive hidden truth, NPC private beliefs, future scheduled events, unresolved proposals, or unobserved offscreen battles. [VERIFIED: docs/WorldForge_Key_NPC_AI_players_latency_memory_budget_full_report_v3.md:934-965]

5. A false player claim creates a claim event and possible beliefs, not backend truth. [VERIFIED: docs/WorldForge_Key_NPC_AI_players_latency_memory_budget_full_report_v3.md:1179-1204]
6. Tool success must produce concrete state/event/knowledge/visibility artifacts when the action claims durable consequence. ## Open Questions (RESOLVED)

1. `world_time` representation: use authoritative integer `world_time_minutes` relative to campaign start, with `currentTick` kept as compatibility/read-model metadata during migration. Rich calendar labels can be derived later, but scheduling, due points, elapsed actions, and rollback compare integer minutes. [DECIDED 2026-05-07]
2. `SimulationJob` and `SimulationProposal` storage: store authoritative jobs/proposals in additive SQLite tables. Config JSON may retain compatibility/read-cache values, but it is not the authority for simulation queue, proposal commit/rebase, actor process, command node, or rollback state. [DECIDED 2026-05-07]
3. `simulateOffscreenNpcs` compatibility: do not keep it as an authoritative writer. During migration, existing logic may only be reused behind a no-direct-write proposal adapter or test fixture. Phase 88 acceptance requires detached direct calls to stop mutating state after `done`. [DECIDED 2026-05-07]
4. Key actor eligibility: `npcs.tier === "key"` gets a durable KeyActorProcess by default. Persistent NPCs may be promoted into actor-process simulation by explicit process row/promotion rule. Temporary/support NPCs do not receive heavy process simulation unless promoted. [DECIDED 2026-05-07]
5. Latency budget: preserve the report SLOs as product diagnostics, not kill-switches. Normal interactive turns target p50 <= 5 min, p90 <= 8 min, and <= 4 serialized LLM decision groups. Heavy turns target p95 <= 10 min and <= 5 serialized groups. A valid long turn must not be aborted by arbitrary duration caps; trace violations drive scheduling/context/parallelism fixes. [DECIDED 2026-05-07]
