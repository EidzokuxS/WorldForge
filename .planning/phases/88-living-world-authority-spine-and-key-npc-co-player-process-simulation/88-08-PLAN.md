# 88-08 Plan: Faction Command Networks

## Goal

Replace omniscient faction macro ticks with faction state plus explicit command actors, command nodes, units, standing orders, reports, resources, and communication latency.

## Scope

- Add faction command network data.
- Add command-node actor frame and decision runner.
- Add report inboxes, standing orders, operations, unit/resource ledgers, and communication delays.
- Refactor current faction tool path so abstract faction cognition is no longer the final agency model.

## Acceptance

- Faction action requires an arrived report, standing order trigger, or command actor observation.
- Command decisions are bounded by authority, resources, communication, unit availability, target scope, and operation preconditions.
- A faction cannot instantly know a local crime, hidden lie, or distant attack without a propagation route.

- Typecheck.
