# 88-01 Plan: Authority Spine Data Model

## Goal

Add the authoritative persistence and type contracts required for world version, world time, event provenance, ToolResult, simulation jobs, proposals, actor process state, and rollback-aware commits.

## Scope

- Add durable `WorldClock` / version state.
- Add additive tables or equivalent authoritative records for simulation jobs, proposals, actor process state, command/thread ledgers, and authority traces.
- Expand `ToolResult` semantics so every future-relevant mutation can be traced to a backend-validated result.
- Extend snapshot/restore so simulation authority artifacts cannot survive retry/undo/checkpoint restore incorrectly.

## Implementation Notes

- Do not remove `currentTick` immediately. Bridge it into world clock compatibility first.
- Use additive migrations.
- Do not route any new actor/faction autonomy through these tables until stale-version rejection and rollback tests exist.

## Acceptance

- Every authoritative mutation can be tied to campaign id, base version, result version, world time/tick, source entity, event ids, and ToolResult.
- Invalid or stale version writes fail before state mutation.
- Restore cancels or supersedes jobs/proposals/process artifacts from discarded versions.
