## Tasks

1. Add checklist schema with step id, purpose, evidenceRefs, dependency, expectedVisibleEffect, requiredAction kind, status, and optional bounded `candidateToolRequest`.
2. Checklist max size is 6 items. It may include backend-approved refs/candidate refs for grounding. It must reject model-generated backend IDs, state deltas outside tool input, narrator prose, hidden/private forecast terms, and arbitrary executable smuggling.
3. Generate checklist only for tool/combat paths.
4. Add prompt contract that explains the checklist is an auditable GM plan, not execution.
5. Define dependency semantics: checklist order is sequential; if a dependency is skipped/failed, dependents are auto-skipped unless GM revision explicitly changes them within the revision budget.
6. Define requiredAction enum. Oracle may appear only if it represents a new uncertainty not already resolved by Stage 2; otherwise repeated oracle for the same uncertainty is invalid.
7. Add tests for capped size and executable smuggling rejection.
8. Test migration: enumerate ScenePlanner/checklist tests superseded by this plan.

## Acceptance Criteria

- Mutating/combat turns have an inspectable consequence checklist.
- Checklist explains why each step exists.
- Checklist candidate tool requests are untrusted until backend step validation succeeds.
