## Tasks

1. Add RED ordering tests: direct/continue/clarification path skips ScenePlanner/action execution.
2. Keep Oracle conditional and binding.
3. Inject Oracle result explicitly into later checklist/narration context when path is `roll_oracle`.
4. Build no-mutation settled packet directly for non-mutating paths. Delete `buildNoMutationScenePlan` or mark it as a temporary shim with an explicit removal task in this phase.
5. Add path/action consistency audit that does not perform backend semantic NLU. It may verify schema/path invariants and log suspicious cases, but it must not reinterpret raw player prose as backend truth.
6. Preserve and update SSE stage events according to `81-SPEC.md`: `gm-read`, optional `oracle`, optional `gm-action-checklist`, optional `tool-step`, `settled-packet`, `final-narration`.
7. Remove live reliance on vestigial BeatPlan projection.
8. Own `gm-beat-plan` cleanup: remove imports, projections, `playerFacingBeat` references, and delete `backend/src/engine/gm-beat-plan.ts` / its tests if no remaining non-live owner exists.
9. Test migration: enumerate `turn-processor.scene-plan.test.ts`, `chat.scene-plan.test.ts`, and UI stage tests that must survive or change.

## Acceptance Criteria

- Non-mutating paths cannot produce plannedActions.
- Planner/tool executor are not called for direct/continue/clarification.
- Narration still works from settled truth.
- New SSE stage sequences are asserted per path.
