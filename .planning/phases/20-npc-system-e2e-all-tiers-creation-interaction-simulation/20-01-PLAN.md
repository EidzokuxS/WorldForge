**Test sequence -- 8+ turns focused on NPC behavior:**

1. **Setup: Verify NPC baseline**
   - GET /api/campaigns/:id/world -- record all NPCs, their tiers, locations, goals, beliefs
   - Identify which Key NPCs are at player's location (candidates for ticks)
   - Identify which Key NPCs are NOT at player's location (candidates for off-screen sim)
   - Log NPC count per tier: key, persistent, temporary

2. **Turn 1 -- NPC interaction (talk to named NPC):**
   - Get a Key NPC name from world data at player's location
   - Send action: "I approach [NPC name] and ask them about their current plans and goals"
   - Parse SSE stream -- verify narrative mentions the NPC by name
   - Wait 45s for GLM rate limits
   - After turn completes, GET /api/campaigns/:id/world again
   - Compare NPC data: check if the spoken-to NPC's goals, beliefs, or tags changed (NPC tick effect)

3. **Turn 2 -- Provoke spawn_npc:**
   - Send action designed to trigger NPC spawning: "I wander into a crowded area and look for someone suspicious lurking in the shadows"
   - Parse SSE stream -- check state_update events for spawn_npc tool call
   - After turn, GET world data -- check if new temporary-tier NPC appeared
   - Log any new NPCs found (compare against baseline)

4. **Turn 3 -- Another NPC interaction:**
   - If spawn_npc created a new NPC, try talking to it: "I approach the [new NPC description] and demand to know who they are"
   - Otherwise talk to another NPC at location
   - Verify narrative is contextually appropriate
   - After turn, check world data for NPC tick effects

5. **Turn 4-5 -- Advance ticks for off-screen simulation:**
   - Send 2 actions to advance tick counter (off-screen sim fires every 5 ticks per Phase 16 findings)
   - After each turn, check world data for changes in NPCs NOT at player location
   - Log any off-screen NPC state changes (location, goals, beliefs)

6. **Turn 6 -- Combat near NPC (NPC reaction test):**
   - Send combat action: "I draw my weapon and attack [target]"
   - After turn, check if Key NPCs at location reacted (changed goals/beliefs/tags)

7. **Turn 7-8 -- Continue advancing ticks:**
   - Send 2 more actions to further advance tick counter
   - After each turn, compare world data against previous snapshot
   - Track cumulative NPC changes: how many NPCs had any state change across all turns

**NPC tier promotion test (separate from gameplay):**
- Find a temporary or persistent NPC from world data
- POST /api/campaigns/:id/npcs/:npcId/promote with { newTier: "persistent" } (or "key")
- Verify 200 response with old/new tier
- Try promoting downward (key -> temporary) -- verify 400 error
- GET world data and confirm NPC tier changed

**Rate limiting:** 45s cooldown between turns (per Phase 19 pattern).

**Assertions (minimum):**
- All 8 turns complete without SSE error events
- At least 1 turn narrative mentions an NPC by name (interaction worked)
- NPC world data changes observed across turns (ticks are firing)
- Tier promotion succeeds upward, fails downward
- spawn_npc may or may not trigger (LLM discretion) -- log but don't hard-fail
