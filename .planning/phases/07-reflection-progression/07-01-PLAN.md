  <action>
    **reflection-tools.ts**: Create `createReflectionTools(campaignId, npcId)` factory returning 4 AI SDK tools:
    - `set_belief`: inputSchema `z.object({ belief: z.string(), evidence: z.array(z.string()) })`. Execute: load NPC beliefs JSON, append new belief string to array, write back. Keep max 10 beliefs (drop oldest if exceeded).
    - `set_goal`: inputSchema `z.object({ goal: z.string(), priority: z.enum(["short_term", "long_term"]) })`. Execute: parse goals JSON, append to the appropriate array, write back. Max 5 per category.
    - `drop_goal`: inputSchema `z.object({ goal: z.string() })`. Execute: parse goals JSON, remove matching goal from both short_term and long_term arrays (case-insensitive match), write back.
    - `set_relationship`: inputSchema `z.object({ target: z.string(), tag: z.string(), reason: z.string() })`. Execute: call existing `executeToolCall(campaignId, "set_relationship", { entityA: npcName, entityB: target, tag, reason }, 0)` to reuse the validated relationship upsert logic.

    **reflection-agent.ts**: Two exported functions:
    1. `runReflection(campaignId, npcId, tick, judgeProvider, embedderProvider?)`:
       - Load NPC from DB (name, tags, beliefs, goals, unprocessedImportance)
       - Search episodic events involving this NPC via embedTexts([npcName]) then searchEpisodicEvents(vector, tick, 10) — fetch more events (10) for reflection vs normal tick (3)
       - Build system prompt: "You are reflecting on recent experiences as {npcName}. Based on these events, update your beliefs, goals, and relationships. Only make changes that are clearly supported by the evidence."
       - Include current beliefs, goals, and the episodic event texts in the prompt
       - Call `generateText({ model, tools: createReflectionTools(campaignId, npcId), temperature: 0, stopWhen: stepCountIs(3), system, prompt })` using Judge role
       - After LLM call completes, reset `unprocessedImportance` to 0 via `db.update(npcs).set({ unprocessedImportance: 0 }).where(eq(npcs.id, npcId)).run()`
       - Return reflection results (tool calls made) for logging
    2. `checkAndTriggerReflections(campaignId, tick, judgeProvider, embedderProvider?)`:
       - Query all NPCs in campaign with `unprocessedImportance >= 15` (threshold constant `REFLECTION_THRESHOLD = 15`)
       - For each qualifying NPC, call runReflection sequentially (same pattern as tickPresentNpcs)
       - Log but never throw — failures are non-blocking
       - Return array of results

    Use `createLogger("reflection-agent")` for logging. Follow exact patterns from npc-agent.ts.
