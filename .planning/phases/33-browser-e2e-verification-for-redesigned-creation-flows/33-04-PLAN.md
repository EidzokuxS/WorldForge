    Use one of the campaigns created in Plans 02 or 03 (or create a new one if needed).

    Using PinchTab, execute the character creation flow:

    1. **Navigate to character creation**:
       - From the campaign's review page, find and click "Continue to Character" or equivalent button
       - OR navigate directly to `http://localhost:3000/campaign/[id]/character` (replace [id] with actual campaign ID)
       - Confirm the character page renders inside the shell
       - Confirm page header shows "Character" or "Character Authoring" eyebrow
       - Confirm character creation input modes are visible (describe, generate, import V2)

    2. **Create character via description parsing** (D-01 flow 2):
       - Select the "Describe" mode (or whichever mode takes free-text description)
       - Enter character description: "A grizzled former soldier turned wandering healer. Tall, scarred, with grey-streaked dark hair. Carries a worn medical kit and a short blade for self-defense. Skilled in herbalism and field medicine, but haunted by memories of the war."
       - Submit the description
       - Wait for LLM parsing (real GLM call, up to 60s)
       - Confirm the CharacterCard shows parsed fields:
         - Name (LLM-generated from description)
         - Race/species
         - Appearance details
         - Tags (skills, traits)

    3. **Verify character card editing**:
       - In the character card, try editing a field (e.g., change the character name)
       - Confirm the edit is reflected in the card
       - Verify tags can be added or removed if tag editor is visible

    4. **Persona template selection** (if available in UI):
       - Look for persona template selector in the workspace
       - If templates are listed, select one and verify it updates the character draft
       - If no templates are available for this campaign, confirm the selector shows empty state gracefully

    5. **Start conditions and loadout preview** (if visible in workspace):
       - Look for start condition or loadout section
       - Verify it shows relevant starting information (starting location, initial items)
       - If start condition resolution is a separate step, trigger it

    6. **Save character and verify game handoff**:
       - Find and click the "Save" or "Begin Adventure" or "Start Game" button
       - Confirm the save triggers a backend call (POST /api/worldgen/save-character)
       - Confirm redirect to `/game` page (or `/campaign/[id]/game`)
       - Confirm the game page loads (narrative area visible, not an error page)
       - Confirm the game page is OUTSIDE the non-game shell (no sidebar)

    Per D-07: Fix all bugs inline.
    Per D-05: Real LLM calls only (GLM provider).
    Per D-06: Retry up to 3 times on LLM failure.

    Common character creation bugs to watch for:
    - ParsedCharacter fields not mapping to CharacterCard display
    - Save button disabled or not responding
    - Missing campaign ID in route causing 404
    - Game page not loading after save (redirect URL wrong)
    - Character data not persisting to DB (game page shows no character)
