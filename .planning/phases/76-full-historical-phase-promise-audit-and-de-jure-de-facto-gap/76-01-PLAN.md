<interfaces>
Expected final audit table shape:
```markdown
| Audit Key | Phase Source | Phase # | Title | Material Promise | Evidence Checked | Classification | Risk | Disposition | Code/Tests/Docs Change |
|---|---|---:|---|---|---|---|---|---|---|
```

Row granularity and parser contract:
- Minimum one row per expected phase key; use multiple rows for a phase when materially distinct promises need different evidence or classifications.
- `Audit Key` must be globally unique. Use keys like `37-current:runtime-loop`, `17-legacy:worldgen-e2e`, or `19.1-legacy:cleanup`.
- Column order is fixed exactly as shown above. Markdown pipe characters inside cells must be escaped as `\|`; unescaped pipes that change column count are invalid.
- Empty cells are invalid. Use `n/a` only when the schema explicitly allows no value.
- Every slice and final audit must include a `## Structured Audit Rows` appendix containing one JSON object per line in a fenced `jsonl` block. The validator parses the structured appendix as canonical and checks that the Markdown table mirrors the same `Audit Key` set.
- Path-like evidence values in markers such as `source:`, `test:`, `frontend:`, and `verification:` must exist when the value starts with `.planning/`, `backend/`, `frontend/`, `shared/`, `docs/`, or `tasks/`, or contains a slash/backslash. Route names such as `route:POST /api/chat` are not filesystem paths.

Allowed classifications:
`verified-current`, `stale-unwired`, `partial`, `superseded`, `deprecated`, `follow-up`, `not-applicable`, `needs-human-UAT`.

Required evidence markers:
`source:`, `test:`, `route:`, `runtime:`, `frontend:`, `verification:`, `supersession:`, `deprecation:`, `uat:`.

For `verified-current`, require at least one current live marker from `source:`, `test:`, `route:`, `runtime:`, or `frontend:`. Do not accept `SUMMARY`, checkbox, or roadmap completion evidence alone.
