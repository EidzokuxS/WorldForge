# Phase 19.1 Plan 01: Remove Gameplay Fallbacks Summary

**Generic withModelFallback<T> utility replacing all silent gameplay degradations (Oracle coin flip, empty Storyteller narrative, swallowed lore errors) with Fallback role retry**

## Performance

- **Duration:** 8 min
- **Started:** 2026-03-20T18:35:16Z
- **Completed:** 2026-03-20T18:43:14Z
- **Tasks:** 2
- **Files modified:** 9

## Accomplishments
- Created generic withModelFallback utility and resolveFallbackProvider helper for consistent primary->fallback retry across all AI systems
- Oracle no longer returns 50% coin flip on failure -- retries with Fallback role, then throws if both fail
- Storyteller retries with Fallback model when primary stream fails before any narrative text is sent
- Lore extraction adds a Fallback role retry after existing 3-attempt strategy (2x full + 1x reduced) exhausts
- Scaffold generator propagates lore extraction errors to SSE caller instead of silently returning empty loreCards
- Premise retrieval in post-turn image context throws naturally instead of silently returning empty string

## Decisions Made
- withModelFallback is the single utility for all fallback logic -- Oracle, Storyteller, and Lore all route through it
- Storyteller fallback only triggers if no narrative text has been sent to client (narrativeStarted === false) -- partial stream failures are rare and propagate as hard errors
- Lore extraction uses withModelFallback with a guaranteed-failure primary thunk after all primary retries exhaust, ensuring consistent logging and code paths
- Scaffold generator now propagates lore extraction errors instead of returning empty array -- the SSE progress will report the error to the user
