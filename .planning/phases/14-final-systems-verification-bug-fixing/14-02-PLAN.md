**Fix in game/page.tsx:**

In the `submitAction` function's `parseTurnSSE` callbacks, change `onStateUpdate` to trigger a world data refresh:

```typescript
onStateUpdate: (update) => {
  // Refresh world data on any state change (movement, spawn, item transfer, etc.)
  if (activeCampaign) {
    void refreshWorldData(activeCampaign.id);
  }
},
```

Apply the same fix in `handleRetry`'s `parseTurnSSE` callbacks — it also has `onStateUpdate: () => {}`.

This means world data refreshes on EVERY state_update event during streaming, not just at the end. This is fine because:
- state_update events are infrequent (1-3 per turn)
- `refreshWorldData` is a simple GET to `/api/campaigns/:id/world`
- Multiple rapid calls are harmless (last one wins via React setState)

Keep the `onDone` refresh as well — it serves as a final sync to catch any tool calls that fired at the very end of streaming.

Do NOT debounce — the calls are infrequent enough that debouncing adds complexity without benefit.

The lore extraction fails silently with provider errors, producing 0 lore cards. The caller (scaffold-generator pipeline) already has a try/catch, but the user gets no feedback about WHY it failed.

**Fix in lore-extractor.ts:**

1. Wrap the `generateObject` call in a retry loop (max 2 attempts). Some providers fail transiently on large structured outputs.

```typescript
export async function extractLoreCards(
  scaffold: WorldScaffold,
  role: ResolvedRole
): Promise<ExtractedLoreCard[]> {
  const context = formatScaffoldContext(scaffold);
  const prompt = `...`; // unchanged

  const MAX_RETRIES = 2;
  let lastError: Error | null = null;

  for (let attempt = 1; attempt <= MAX_RETRIES; attempt++) {
    try {
      const result = await generateObject({
        model: createModel(role.provider),
        schema: loreExtractionSchema,
        prompt,
        temperature: role.temperature,
        maxOutputTokens: role.maxTokens,
      });
      return result.object.loreCards;
    } catch (error) {
      lastError = error instanceof Error ? error : new Error(String(error));
      log.warn(`Lore extraction attempt ${attempt}/${MAX_RETRIES} failed: ${lastError.message}`);
      if (attempt < MAX_RETRIES) {
        // Brief pause before retry
        await new Promise(r => setTimeout(r, 2000));
      }
    }
  }

  // If all retries failed, try with a smaller schema (min 10 instead of 20)
  try {
    log.info("Attempting lore extraction with reduced card count (10-30)");
    const reducedSchema = z.object({
      loreCards: z.array(loreCardSchema).min(10).max(30),
    });
    const result = await generateObject({
      model: createModel(role.provider),
      schema: reducedSchema,
      prompt,
      temperature: role.temperature,
      maxOutputTokens: role.maxTokens,
    });
    return result.object.loreCards;
  } catch {
    // All attempts failed — throw with helpful message
    throw new Error(
      `Lore extraction failed after ${MAX_RETRIES} retries: ${lastError?.message ?? "unknown error"}. ` +
      `Check that the Generator provider (${role.provider.model}) supports structured JSON output.`
    );
  }
}
```

2. Add `import { createLogger } from "../lib/index.js"` and create a logger:
```typescript
const log = createLogger("lore-extractor");
```

This gives 3 total attempts: 2 at full size (20-60 cards), then 1 at reduced size (10-30 cards). The reduced attempt helps with providers that struggle with large structured outputs.
