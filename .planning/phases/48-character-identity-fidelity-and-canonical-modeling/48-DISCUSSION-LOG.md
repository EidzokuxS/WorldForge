# Phase 48 Discussion Log

**Date:** 2026-04-12
**Phase:** 48 — Character Identity Fidelity & Canonical Modeling

## Topics Covered

### 1. What a character must fundamentally contain
- Decision: important characters need a real identity core, not just a short summary plus tags.
- User direction: a character should preserve the traits that would actually affect behavior, goals, and reactions, especially for imported and canonical characters.

### 2. Whether canonical characters need a stricter path
- Decision: yes, key and canonical characters should use a stricter fidelity path than generic characters.
- User direction: how those characters feel strongly shapes how the game feels, but they still must remain people rather than rigid “roles.”

### 3. What layers define canonical character truth
- Decision: use three layers:
  - base facts,
  - behavioral core,
  - live campaign dynamics.
- User direction: the first two define who the character is; the third lets them live and change in the run.

### 4. Where canonical truth should come from
- Decision: use canon-facing sources for facts, community cards as secondary help for feel and behavioral cues, and synthesize the final structured model inside WorldForge.
- User direction: do not blindly trust one card or reduce the character to a dry wiki summary.

### 5. How much canonical characters may change
- Decision: preserve the internal core, but allow justified change over time.
- User direction: characters are not carved in stone; different circumstances can produce different conclusions, as long as the change is earned and not instant personality replacement.

### 6. Whether richer identity should exist only for key characters
- Decision: improve the baseline model for everyone, then add an additional fidelity layer for key/canonical characters.
- User direction: do not build two completely different character systems, but do give important characters more depth and continuity.

## Outcome

Phase 48 should be planned as a rebuild of runtime character identity around a stronger shared model, with additional fidelity and continuity for key/canonical characters and explicit support for justified long-term change without flattening or role-locking them.
