# 80-03 PLAN: Scoped Forecast Excerpts For GM Prompts

## Scope

Expose only prompt-safe forecast excerpts to local GM planning. Remote/offscreen/private forecast entries may exist in storage, but must not appear in a local turn prompt unless surfaced as local or player-known context.

Review amendment: scoped forecast excerpt is mandatory input to GM TurnDecision and later BeatPlan on every turn, even when empty. This plan must not depend on BeatPlan implementation from 80-04; BeatPlan-specific prompt tests move to 80-04.

## Tasks

### Task 1: RED Remote Forecast Leak Tests

Add tests where:

- stored forecast includes `Forest Outpost` and a hidden/offscreen actor;
- player is in a Shibuya local frame;
- GM decision prompt and ScenePlanner prompt do not include remote private terms;
- the remote terms are absent even when they are not already listed in `scenePacket.safety.forbiddenActorLabels`;
- player-known hints can appear only as anonymized/local pressure, not omniscient truth.

### Task 2: GREEN Scoped Forecast Excerpt

Implement forecast scoping helpers that:

- include entries grounded in local candidates/current scene/player-known facts;
- omit remote/private entries by default;
- can produce opaque player-known pressure hints without leaking exact hidden refs;
- reuse Phase 79 model-facing prompt safety for defense in depth.

### Task 3: Prompt Formatting

Add forecast excerpt formatting helpers for GM decision and ScenePlanner without changing the final narration boundary yet. The formatted excerpt must be concise and bounded. The full forecast object must never be passed to prompt assemblers.

## Acceptance Criteria

- Forecast storage can contain offscreen pressure without local prompt leakage.
- Scoping is deterministic and test-covered.
- GM TurnDecision always receives a scoped forecast excerpt, even when empty.
- Prompt builders receive only scoped forecast excerpts, not full forecasts.
