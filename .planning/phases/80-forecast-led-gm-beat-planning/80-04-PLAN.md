## Tasks

### Task 1: RED BeatPlan Schema Tests

Create `gm-beat-plan.test.ts` proving:

- BeatPlan requires beat intent, why-now, local focus, pacing/tension posture, reveal budget, forecast influence refs, and tool posture;
- direct/no-roll turns can produce a non-mutating BeatPlan;
- BeatPlan rejects direct deltas and executable ScenePlan/tool payloads;
- BeatPlan rejects nested executable smuggling such as `metadata.toolInput`, `revealBudget.durableEvent`, `plannedActions`, `payload`, and `narratorFacts`;
- BeatPlan records non-forcing guidance so forecasts create pressure without railroading player agency;
- player-facing formatter excludes private rationale.

### Task 2: GREEN BeatPlan Schema And Formatters

Implement:

- strict `gmBeatPlanSchema`;
- `validateBeatPlanForFrame`;
- `formatBeatPlanForScenePlanner`;
- `formatBeatPlanForNarrator`;
- explicit private vs player-facing field separation.

### Task 3: GREEN `runGmBeatPlanner`

Implement a safe structured-output call that receives:

- raw player action;
- current GM decision;
- model-facing local scene packet;
- scoped forecast excerpt;
- allowed tool posture/candidate summary.

Prompt contract must tell the model what to output and what not to output. It must not ask for runtime tool input payloads. It must explain why this beat now, what happens if the player does nothing, and what alternatives must remain open to the player.

### Task 4: RED/GREEN Prompt Leak And Posture Tests

Add tests proving:

- BeatPlan prompt receives only scoped forecast excerpts, not full forecasts;
- private forecast refs such as `Forest Outpost` are absent unless local/player-known;
- BeatPlan formatter for ScenePlanner carries coarse posture/tool-category guidance but no executable payload;
- BeatPlan formatter for Narrator carries only settled player-facing guidance.

## Acceptance Criteria

- BeatPlan exists as a separate advisory artifact.
- BeatPlan cannot mutate state or smuggle tool payloads.
- BeatPlan is a hard intent contract for downstream tool posture/category.
- BeatPlan has player-facing and private formatting boundaries.
