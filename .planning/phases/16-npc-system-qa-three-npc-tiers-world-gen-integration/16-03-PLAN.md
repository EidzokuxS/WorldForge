**Setup:**
1. Load the campaign via `POST http://localhost:3001/api/campaigns/{id}/load`.
2. Get active campaign state via `GET http://localhost:3001/api/campaigns/active`.
3. Fetch world data via `GET http://localhost:3001/api/campaigns/{id}/world`.

**Test A: Key NPC Autonomous Tick**

1. Send a player action via `POST http://localhost:3001/api/chat` with:
   ```json
   {"action": "I look around the area and observe what everyone is doing."}
   ```
   Stream the SSE response and capture all events.

2. After the response completes, check backend logs for NPC tick activity:
   - Look for log lines from `npc-agent` logger: "tick complete", "tool call(s)"
   - If possible, run the backend with visible stdout/stderr to capture logs
   - Alternatively, check if the SSE response includes `npc_actions` event type

3. Verify from the response:
   - The narrative mentions NPCs present at the location (by name)
   - NPC actions/dialogue may appear in the narrative or as separate events
   - `state_updates` event may include NPC-related changes

4. Send 2-3 more player actions to trigger multiple NPC tick cycles:
   - `{"action": "I try to talk to {key_npc_name}."}`
   - `{"action": "I ask {key_npc_name} about their goals."}`
   Verify NPC responses are contextually appropriate.

**Test B: spawn_npc Tool (Minor NPCs)**

   ```json
   {"action": "I look for a local merchant to buy supplies from."}
   ```
   or
   ```json
   {"action": "I approach a random stranger on the street and ask for directions."}
   ```

2. Check the SSE response for `state_updates` events that include `spawn_npc` tool calls.

3. After the turn, re-fetch world data and check if a new NPC appeared:
   - `GET http://localhost:3001/api/campaigns/{id}/world`
