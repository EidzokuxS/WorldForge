  <action>
Test the scaffold generation pipeline's NPC output using the existing "Polish Test" campaign or by creating a fresh test campaign via API. Steps:

1. **List campaigns** via `GET http://localhost:3001/api/campaigns` -- find the "Polish Test" campaign or identify a campaign with completed world generation.

2. **Load the campaign** via `POST http://localhost:3001/api/campaigns/{id}/load`.

3. **Fetch world data** via `GET http://localhost:3001/api/campaigns/{id}/world` and verify the response contains an `npcs` array.

4. **For each NPC in the response, verify:**
   - `id` is a valid UUID string
   - `name` is a non-empty string
   - `tier` is one of: `"temporary"`, `"persistent"`, `"key"`
   - `persona` is a non-empty string (at least 20 chars for key NPCs)
   - `tags` is a valid JSON array (parse it, confirm it's an array of strings)
   - `goals` is a valid JSON object with `short_term` and `long_term` arrays
   - `beliefs` is a valid JSON array of strings
   - `currentLocationId` is either null or a valid UUID that matches one of the location IDs from the same response
   - `createdAt` is a valid timestamp number

5. **Cross-reference NPCs with locations:** For each NPC that has a `currentLocationId`, verify that ID exists in the `locations` array from the world data response.

6. **Cross-reference NPCs with factions:** Check that NPC tags or persona mention faction names that exist in the `factions` array.

7. **Count tier distribution:** Log how many NPCs are in each tier (key/persistent/temporary). Scaffold generation should produce at least 3 key NPCs.

8. **Log all results** as structured output with PASS/FAIL per check.

Use `curl` commands against `http://localhost:3001` for all API calls. Parse JSON responses with `jq` or inline Node.js scripts.

   }
   ```

   }
   ```

3. **Research + Generate NPC** -- `POST http://localhost:3001/api/worldgen/research-character`:
