**Prerequisites:** Ensure backend is running on :3001. Use an existing campaign or create one programmatically (POST /api/campaigns, then scaffold generation, then character save).

**Test sequence — 5+ turns via API:**

1. **Turn 1 — Exploration action:** Send "I look around and examine my surroundings carefully" via POST /api/chat with `{ message: "...", settings: { ... } }` (inline settings with GLM provider for Judge and Storyteller). Parse SSE stream and verify:
   - `oracle_result` event received with fields: `chance` (number 0-100), `roll` (number 1-100), `tier` (one of "strong_hit", "weak_hit", "miss")
   - `narrative` events received with non-empty text
   - `quick_actions` event received with array of 3-5 actions
   - `done` event received signaling turn completion
   - Log chance/roll/tier for quality assessment

2. **Turn 2 — NPC interaction:** Send "I approach the nearest person and ask about recent events in town" (or contextually appropriate action). Verify same SSE event structure. Check if Storyteller calls any tools (add_tag, set_relationship, log_event).

3. **Turn 3 — Combat initiation:** Send "I draw my weapon and attack the nearest hostile creature" (or similar combat action). Verify:
   - Oracle gives reasonable probability (not 95% for unprovoked attack)
   - If outcome involves damage, `state_update` event shows `set_condition` tool execution
   - After turn, GET /api/campaigns/:id/world and check player HP if combat happened

4. **Turn 4 — Movement:** Send "I leave this place and travel to [adjacent location from world data]" (read available locations from GET /api/campaigns/:id/world first). Verify:
   - If move_to tool is called, player location changes
   - GET /api/campaigns/:id/world after turn shows new current location

5. **Turn 5 — Quick action replay:** Take one of the quick_actions from a previous turn and send it as the next action. Verify the turn processes normally.

**SSE parsing:** Read the response body as text stream. Each SSE event is formatted as `event: {type}\ndata: {json}\n\n`. Parse each event type (oracle_result, narrative, state_update, quick_actions, done, error).

**Quality assessment per turn:**
- Oracle probability reasonable for action type (exploration 70-90%, combat 30-70%, absurd <20%)
- Narrative text is coherent, matches outcome tier
- Tools called when narratively appropriate
- No errors in stream

**Rate limiting:** Add 30-second cooldown between turns if GLM rate limits are hit (per Phase 18 pattern).

**Assertions (minimum):**
- All 5 turns complete without error events
- Every turn has oracle_result with valid chance/roll/tier
- Every turn has narrative text > 50 characters
- At least 3 of 5 turns have quick_actions
- At least 1 turn has a state_update from tool execution
- Overall narrative quality >= 4.5/5 (human assessment in summary)
