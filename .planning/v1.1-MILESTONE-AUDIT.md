# Milestone v1.1 Audit

## Summary

Milestone `v1.1` is not ready for closeout.

Most phase seams are real and well tested, but the audit found the same class of problem that already surfaced in Phase 48: some route matrices were treated as complete when one or more real entry paths were still outside the verification boundary.

## Confirmed Implementation Gaps

### 1. Phase 45 / Phase 47: legacy `POST /api/chat` still bypasses the repaired gameplay path

Evidence:
- [backend/src/routes/chat.ts](R:\Projects\WorldForge\backend\src\routes\chat.ts)
- [backend/src/ai/storyteller.ts](R:\Projects\WorldForge\backend\src\ai\storyteller.ts)
- [45-VERIFICATION.md](R:\Projects\WorldForge\.planning\phases\45-authoritative-scene-assembly-and-start-of-play-runtime\45-VERIFICATION.md)
- [47-VERIFICATION.md](R:\Projects\WorldForge\.planning\phases\47-storyteller-output-quality-and-anti-slop-prompting\47-VERIFICATION.md)

What is wrong:
- The legacy plain-text `POST /api/chat` route is still live.
- It uses `callStoryteller()` with `worldPremise` and old chat history rather than the Phase 45 authoritative scene assembly and the Phase 47 storyteller preset/guard path.
- Phase 45 and Phase 47 verification prove `/chat/action`, `/chat/opening`, and `/game`; they do not close this legacy entry path.

Why it matters:
- Any caller still using the legacy route can bypass `SCEN-01` and `WRIT-01`.
- This is not just stale documentation; it is a real alternate runtime path.

### 2. Phase 49: `/lookup` and `/compare` do not survive reload/history

Evidence:
- [backend/src/routes/chat.ts](R:\Projects\WorldForge\backend\src\routes\chat.ts)
- [frontend/app/game/page.tsx](R:\Projects\WorldForge\frontend\app\game\page.tsx)
- [49-VERIFICATION.md](R:\Projects\WorldForge\.planning\phases\49-search-grounding-and-in-game-research-semantics\49-VERIFICATION.md)

What is wrong:
- `/api/chat/lookup` streams `lookup_result` and `done`, but does not append the lookup exchange to persisted chat history.
- `/game` restores its log from `/api/chat/history`.
- That means live lookup/compare appears in-session, but not after reload.

Why it matters:
- Phase 49 presents lookup as part of the normal game log contract.
- Right now that contract is session-only, not reload-stable.

### 3. Phase 49 / Phase 52: draft-backed NPC manual edits can be dropped on save/load

Evidence:
- [frontend/components/world-review/npcs-section.tsx](R:\Projects\WorldForge\frontend\components\world-review\npcs-section.tsx)
- [backend/src/routes/schemas.ts](R:\Projects\WorldForge\backend\src\routes\schemas.ts)
- [backend/src/worldgen/scaffold-saver.ts](R:\Projects\WorldForge\backend\src\worldgen\scaffold-saver.ts)

What is wrong:
- The world-review UI edits top-level scaffold NPC fields like `name`, `persona`, `tags`, and `goals`.
- For draft-backed NPCs, `saveEditsSchema` rebuilds from `input.draft` and only overrides `locationName`, `factionName`, and `tier`.
- Persistence then prefers `npc.draft` again in `scaffold-saver.ts`.

Why it matters:
- Generated/imported/researched NPCs are increasingly draft-backed.
- Those NPCs can appear editable in World Review while silently discarding some edits after save/load.

### 4. Phase 50 cross-phase fallout: settings/provider persistence fragility surfaced after closeout

Evidence:
- [backend/src/settings/manager.ts](R:\Projects\WorldForge\backend\src\settings\manager.ts)
- [frontend/app/(non-game)/settings/page.tsx](R:\Projects\WorldForge\frontend\app\(non-game)\settings\page.tsx)
- [frontend/lib/use-settings.ts](R:\Projects\WorldForge\frontend\lib\use-settings.ts)
- [50-VERIFICATION.md](R:\Projects\WorldForge\.planning\phases\50-gameplay-text-presentation-and-rich-readability\50-VERIFICATION.md)

What is wrong:
- After Phase 50, the settings path was still vulnerable to silent destructive reset and stale shared-build mismatch around `ui.showRawReasoning`.
- This has already been fixed in code, but it surfaced after the phase was treated as complete.

Why it matters:
- It is a closed defect, but it proves the milestone still had at least one late cross-phase integration miss outside the original verification boundary.
- Closeout should reflect this defect history instead of pretending the phase was clean on first verification.
